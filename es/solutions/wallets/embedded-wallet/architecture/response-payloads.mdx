Al interactuar con la API de Sequence Embedded Wallet, cada carga de intención generará una carga de respuesta correspondiente. Esto podría ser una respuesta exitosa o una solicitud de una acción diferente.

## Estructura General de Respuesta

Independientemente del tipo específico de respuesta, todas las respuestas tienen una estructura estándar con los siguientes campos:

| Nombre | Tipo   | Descripción                                             | Opcional |
| ------ | ------ | ------------------------------------------------------- | -------- |
| code   | string | Identifica el tipo de respuesta                         | No       |
| data   | object | Contiene el contenido definido por el tipo de respuesta | No       |

## Tipos de Respuestas

### Respuesta de Recibo de Transacción

Activado tras la ejecución exitosa de una [send transaction intent](/solutions/wallets/embedded-wallet/architecture/action-payloads#send-transaction-payload). El código para esta respuesta es `transactionReceipt`.

El `data` campo contiene:

| Nombre        | Tipo                                                                                                                                        | Descripción                                        | Opcional |
| ------------- | ------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------- | -------- |
| txHash        | string                                                                                                                                      | Hash de la transacción de Ethereum                 | No       |
| metaTxHash    | string                                                                                                                                      | Hash de la meta transacción                        | No       |
| request       | [SendTransactionPayload](https://docs.sequence.xyz/solutions/wallets/embedded-wallet/architecture/action-payloads#send-transaction-payload) | El paquete de solicitud original de la transacción | No       |
| receipt       | MetaTxnReceipt                                                                                                                              | Recibo de la meta transacción                      | No       |
| nativeReceipt | any                                                                                                                                         | Recibo nativo de la transacción de Ethereum        | Sí       |
| simulations   | SimulateResult\[]                                                                                                                           | Array de resultados de simulación                  | Sí       |

`MetaTxnReceipt`:

| Nombre       | Tipo                 | Descripción                                         | Opcional |
| ------------ | -------------------- | --------------------------------------------------- | -------- |
| id           | string               | Identificador único para el recibo                  | No       |
| status       | string               | Estado de la transacción                            | No       |
| revertReason | string \| null       | Motivo de la reversión de la transacción, si existe | Sí       |
| index        | number               | Número de índice del recibo                         | No       |
| logs         | MetaTxnReceiptLog\[] | Array de registros para la meta transacción         | No       |
| receipts     | MetaTxnReceipt\[]    | Array de recibos de meta transacciones              | No       |
| txnReceipt   | string               | Representación en cadena de la transacción          | No       |

`MetaTxnReceiptLog`:

| Nombre           | Tipo      | Descripción                                                           | Opcional |
| ---------------- | --------- | --------------------------------------------------------------------- | -------- |
| address          | string    | La dirección que generó el registro                                   | No       |
| topics           | string\[] | Lista de temas asociados con el registro                              | No       |
| data             | string    | Datos enviados con el registro                                        | No       |
| blockNumber      | number    | El número de bloque donde se emitió este registro                     | No       |
| transactionHash  | string    | Hash de la transacción que generó el registro                         | No       |
| transactionIndex | number    | Índice de la transacción en el bloque                                 | No       |
| blockHash        | string    | Hash del bloque que contiene el registro                              | No       |
| logIndex         | number    | Índice del registro en el bloque                                      | No       |
| removed          | boolean   | Si el registro fue eliminado debido a una reorganización de la cadena | No       |

`SimulateResult`:

| Nombre    | Tipo           | Descripción                                | Opcional |
| --------- | -------------- | ------------------------------------------ | -------- |
| executed  | boolean        | Si la simulación fue ejecutada             | No       |
| succeeded | boolean        | Si la simulación tuvo éxito                | No       |
| result    | string \| null | El resultado de la simulación              | Sí       |
| reason    | string \| null | Razón del fallo, si la simulación falló    | Sí       |
| gasUsed   | number         | Cantidad de gas utilizado en la simulación | No       |
| gasLimit  | number         | Límite de gas para la simulación           | No       |

### Respuesta de Transacción Fallida

Activado cuando falla la ejecución de un [send transaction intent](/solutions/wallets/embedded-wallet/architecture/action-payloads#send-transaction-payload). El código para esta respuesta es `transactionFailed`.

El `data` campo contiene:

| Campo       | Tipo                                                                                                                                    | Descripción                                                    |
| ----------- | --------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------- |
| error       | string                                                                                                                                  | El mensaje de error que describe por qué falló la transacción. |
| request     | [SendTransactionPayload](https://docs.sequence.xyz//solutions/embedded-wallet/architecture/response-payloads/#send-transaction-payload) | El paquete de transacciones que se intentaron.                 |
| simulations | SimulateResult\[]                                                                                                                       | Resultados de los intentos de simulación de transacciones.     |

`SimulateResult`:

| Nombre    | Tipo           | Descripción                                | Opcional |
| --------- | -------------- | ------------------------------------------ | -------- |
| executed  | boolean        | Si la simulación fue ejecutada             | No       |
| succeeded | boolean        | Si la simulación tuvo éxito                | No       |
| result    | string \| null | El resultado de la simulación              | Sí       |
| reason    | string \| null | Razón del fallo, si la simulación falló    | Sí       |
| gasUsed   | number         | Cantidad de gas utilizado en la simulación | No       |
| gasLimit  | número         | Límite de gas para la simulación           | No       |

### Respuesta de Mensaje Firmado

Activado después de un [sign message intent](/solutions/wallets/embedded-wallet/architecture/action-payloads#sign-message-payload). El código es `signedMessage`.

El `data` campo contiene:

| Nombre    | Tipo   | Descripción        | Opcional |
| --------- | ------ | ------------------ | -------- |
| message   | string | El mensaje firmado | No       |
| signature | string | La firma           | No       |

#### Ejemplo

```json
{
  "code": "signedMessage",
  "data": {
    "message": "Hello World",
    "signature": "0x1234567890abcdef1234567890abcdef1234567890abcdef1234567890abcdef1234567890abcdef1234567890abcdef1234567890abcdef1234567890abcdef1234567890abcdef1234567890abcdef1234567890abcdef"
  }
}
```

#### Cómo Detectar

```ts
import { payloads } from '@0xsequence/waas';

const res = ... // response from the Embedded Wallet API

if (payloads.responses.isSignedMessageResponse(res)) {
  // It's a signed message
}
```

### Respuesta de Sesión Abierta

Ocurre después de ejecutar exitosamente un [open session intent](/solutions/wallets/embedded-wallet/architecture/action-payloads#open-session-payload). El código es `sessionOpened`.

El `data` campo contiene:

| Nombre    | Tipo   | Descripción                            | Opcional |
| --------- | ------ | -------------------------------------- | -------- |
| sessionId | string | ID de sesión (similar a una dirección) | No       |
| wallet    | string | La dirección de la wallet Sequence     | No       |

#### Ejemplo

```json
{
  "code": "sessionOpened",
  "data": {
    "sessionId": "0xBdeAfd3ee0550a73f615D2BaEA2E3D6d48FC857D",
    "wallet": "0x44c2Ea42Fb006a613a7b41D142576eD6B5055902"
  }
}
```

#### Cómo Detectar

```ts
import { payloads } from '@0xsequence/waas';

const res = ... // response from the Embedded Wallet API

if (payloads.responses.isOpenSessionResponse(res)) {
  // It's a session opened
}
```

### Respuesta de Validación Requerida

Algunas acciones pueden requerir validación adicional. En tales casos, el código de respuesta es `validationRequired`.

El `data` campo contiene:

| Nombre    | Tipo   | Descripción                            | Opcional |
| --------- | ------ | -------------------------------------- | -------- |
| sessionId | string | ID de sesión (similar a una dirección) | No       |

#### Cómo Detectar

```ts
import { payloads } from '@0xsequence/waas';

const res = ... // response from the Embedded Wallet API

if (payloads.responses.isValidationRequiredResponse(res)) {
  // Validation is required
}
```

### Nota

Tanto `sendTransaction` como `signMessage` intents podrían requerir validación. Siempre verifica si hay una respuesta `validationRequired` antes de asumir que has recibido un recibo de transacción o un mensaje firmado.

```ts
import { payloads } from '@0xsequence/waas';

const res = ... // response from the Embedded Wallet API

if (payloads.responses.isValidationRequiredResponse(res)) {
  // Handle validation
}

if (payloads.responses.isSendTransactionResponse(res)) {
  // It's a transaction receipt
}
```
