Los "Intents" son acciones que puedes usar para interactuar con tu cuenta, sesiones y billetera. Este documento explica cómo funcionan internamente. No te preocupes por las complejidades - los SDKs se encargan de todo eso.

Piensa en un intent como un objeto JSON. Representa la acción que quieres tomar y está firmado con tu clave de sesión. Por ejemplo, si quieres firmar un mensaje, enviarías un intent a la API de WaaS.

## ¿Qué son?

A alto nivel, un intent es simplemente un objeto JSON que especifica la acción que el usuario pretende realizar y está firmado usando su clave de sesión. Por ejemplo, para firmar un mensaje, el cliente enviaría el siguiente intent a la API de WaaS:

```
{
    "version": "0.0.0",
    "issuedAt": 1712775116,
    "expiresAt": 1712775416,
    "name": "signMessage",
    "data": {
      "wallet": "0x7B7D7BA79542584f9AeF539F6696c070a4e1Ced6",
      "network": "1",
      "message": "0x48656c6c6f2c20686176652061206e69636520646179"
    },
    "signatures": [
      {
        "sessionId": "0x01047de46debeeb83e1270b80ae65996ea64ad2c033e330cfffe9b725bf078423140124dce0cfe0cab9a61b385ca74fe77cd5e8a7147bfd87052f334cfbadc45c79a",
        "signature": "0x28ec6ddcdbc0987a99dde19792ff0c9be69ad2be873fb9351353171b74d2df3380f0ffcae2b8cb37a0659e374859420500b4399a2aab0209ee7919aaea575ac7"
      }
    ]
}
```

Analicemos la estructura de un intent.

* **Firmas:** Usualmente, una única firma representa tu sesión. Una sesión es un par de claves creado localmente; la clave pública es parte del `sessionId` y la clave privada permanece en tu dispositivo. Asegúrate de almacenar tu clave de sesión, que proporciona acceso a tu billetera, de forma segura, idealmente en un enclave seguro del dispositivo. Antes de poder usar la sesión, necesitas registrarla con la API de WaaS.
* La firma es una EIP-191 de la representación canónica del intent. Esto significa que es un JSON con claves ordenadas alfabéticamente, sin espacios en blanco, excluyendo el campo `signatures`.
* **Nombre y Datos:** El nombre y los datos cambiarán con cada intent. El nombre identifica el tipo de intent y los datos asociados con él.
* **Marcas de tiempo:** La API verifica las marcas de tiempo unix para determinar el período de validez del intent y para evitar que los intents antiguos se reutilicen en el futuro.
