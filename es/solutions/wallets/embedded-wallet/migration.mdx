## Descripción General

Este documento proporciona orientación para usuarios que migran desde otros proveedores (como Thirdweb, Venly, etc.) a las Billeteras Integradas de Sequence. El enfoque principal recomendado es un patrón de migración Justo a Tiempo (JIT), que ocurre cuando los usuarios inician sesión por primera vez después de la migración.

## Proceso de Migración Justo a Tiempo

El proceso de migración JIT está diseñado para transicionar suavemente a los usuarios desde su proveedor de billetera existente a las Billeteras Integradas de Sequence con una mínima interrupción en su experiencia. Este proceso ocurre automáticamente cuando un usuario inicia sesión por primera vez después de que se ha implementado la migración, donde el proceso se ve de la siguiente manera:

1. **Autenticación de Usuario**
   * Cuando un usuario intenta iniciar sesión, se autentica con su proveedor social preferido o correo electrónico usando el SDK de Sequence Embedded Wallets.
   * Una vez que el usuario está autenticado, los datos del usuario autenticado como dirección de correo electrónico, nombre de usuario u otro identificador único de usuario junto con la dirección de la wallet de Sequence se pasan a tu backend.

2. **Consulta y Actualización de Base de Datos**
   * Actualiza tu base de datos de usuarios con una columna 'sequenceWallet'.
   * Se consulta la base de datos con el identificador único de usuario para recuperar el registro correspondiente del usuario.
   * El sistema verifica si ya existe un 'sequenceWallet' para este usuario. Si es así, no se toma ninguna acción.
   * Si no se encuentra una dirección de Sequence wallet:
     a. La wallet de Sequence proporcionada se inserta en la columna 'sequenceWallet'.
   * Si un usuario no existe en la base de datos, inserta la información proporcionada como un usuario nuevo.

3. **Migración de Activos**
   * Después de que el usuario haya iniciado sesión exitosamente, el sistema verifica si la migración de activos ya ha ocurrido.
   * Si no, se muestra un mensaje al usuario, explicando la necesidad de migrar sus activos desde su wallet anterior a la nueva Sequence Embedded Wallet.
   * El usuario inicia sesión en el proveedor anterior como un evento único, y migra los activos deseados a su nueva wallet de Sequence.
   * Sugerimos usar el Indexer de Sequence para consultar los activos relevantes y mostrarlos junto con las cantidades al usuario para facilitar la migración.

Por favor, ponte en contacto con cualquier pregunta si tu migración es más compleja y estaremos encantados de ayudar.
