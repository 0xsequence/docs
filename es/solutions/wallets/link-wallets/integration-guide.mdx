# Integrando la Vinculación de Wallets

Tiempo para completar: 25-35 minutos

En esta guía, mostraremos cómo iniciar sesión en una [Embedded Wallet](/solutions/wallets/embedded-wallet/overview) usando autenticación tradicional como correo electrónico o inicio de sesión social. Luego, para vincular una dirección de wallet externa usando Web SDK (ahora Web SDK) a tu Embedded Wallet, firmaremos un mensaje que crea un conjunto consultable de direcciones para un perfil de usuario único almacenado fuera de la cadena de forma privada. Finalmente, se profundizará en otras características de la API, como listar el total de wallets vinculadas y eliminar el estado de las wallets vinculadas, ambas usando firmas para preservar la privacidad.

# Usando la API de Vinculación de Wallets

## 1. Integrando el Archivo de Interfaz de API

Para comenzar a usar la API, se integra un archivo de interfaz en la aplicación clonada para que la API de Sequence pueda ser llamada convenientemente.

El [código fuente de la interfaz se puede encontrar aquí](https://github.com/0xsequence-demos/demo-embedded-wallet-linking/blob/master/src/api/api.gen.ts) que luego se instancia con el servidor Sequence existente de la siguiente manera:

```typescript
import { 
    API,            // API interface
    LinkedWallet    // typed class
} from "./api.gen";

const api = new API("https://dev-api.sequence.app", fetch);
```

#### Ejemplos simples usando las funciones de API integradas en el repositorio usando `0xsequence/waas` y `wagmi` paquetes:

1. [`linkWallet(...)`](/solutions/wallets/link-wallets/integration-guide#5-link-an-external-wallet-to-an-embedded-wallet)
2. [`getLinkedWallets(...)`](/solutions/wallets/link-wallets/integration-guide#6-retrieve-linked-wallets)
3. [`removeLinkedWallet(...)`](/solutions/wallets/link-wallets/integration-guide#7-remove-linked-wallet)

## 2. Vincular una wallet externa a una embedded wallet

Para vincular una wallet externa (por ejemplo, EOA, Sequence Universal Wallet, etc.) a una Embedded Wallet, primero necesitamos autenticar al usuario con una embedded wallet que se categoriza como la `parent` wallet. Una vez autenticado, lo autenticaríamos nuevamente usando Web SDK donde el usuario firma un mensaje después de la autenticación para validar la propiedad de esa wallet. Ese mensaje se pasará al `linkWallet` endpoint de la API, con los siguientes argumentos tipados y tipos de retorno (negando los valores opcionales):

```typescript
interface LinkWalletArgs {
  parentWalletAddress: string // Address of the Embedded Wallet
  parentWalletMessage: string // Message from the Embedded Wallet
  parentWalletSignature: string // Signature from Embedded Wallet
  linkedWalletAddress: string // External wallet address you want to link
  linkedWalletMessage: string // Message from the external wallet
  linkedWalletSignature: string // Signature from the external wallet
  signatureChainId: string // ChainId which you want to verify, using 137 in the example.
  linkedWalletType?: string // Optional value to identify what type of wallet is being connected, i.e., 'sequence'
}

interface LinkWalletReturn {
  status: boolean
}

linkWallet(args: LinkWalletArgs, headers?: object, signal?: AbortSignal): Promise<LinkWalletReturn>
```

### Firmar Mensaje para Atestiguar la Propiedad de una Dirección

Se requieren dos firmas: una para la Embedded Wallet firmando la dirección de la wallet Web SDK (wallet vinculada), y la wallet vinculada (desde Web SDK) firmando la dirección de la wallet principal (Embedded Wallet). Puedes firmar cualquier mensaje presentado al usuario, siempre que el mensaje y la firma sean consistentes con lo que se pasa a la API posteriormente. Por ejemplo, usando el siguiente código en React con Wagmi para obtener esta información usando Sequence:

```typescript
import {
  ...
  useSignMessage,
  useAccount,
  ...
} from "wagmi";

function App() {

    const { signMessageAsync } = useSignMessage();
    const { address: kitWalletAddress } = useAccount();
    
    const getSignaturesForLinking = async () => {
        const parentWalletMessage = "linked wallet with address "
        const linkedWalletMessage = "parent wallet with address "
        const parentMessage = parentWalletMessage + kitWalletAddress
        const parentWalletAddress = await sequenceWaas.getAddress()
        const linkingMessage = "Link to " + linkedWalletMessage + parentWalletAddress

        const response: any = await signMessageAsync({
            message: linkingMessage,
        })

        const parentSignatureRes = await sequenceWaas.signMessage({
            message: parentMessage
        })

        const parentSignature = parentSignatureRes.data.signature
        const linkedSignature = response
        const linkedWalletAddress = kitWalletAddress

        return { parentWalletAddress, linkedWalletAddress, parentMessage, linkingMessage, parentSignature, linkedSignature }
    }
}
```

### Usar API para Agregar Carga Útil de Firma y Dirección de Wallet Principal

Llama a la función anterior `getSignaturesForLinking` y desempaqueta la respuesta para pasar las respuestas a la función `api.linkWallet`:

```typescript
function App(){
    const connections = useConnections();
    ...
    const linkWallet = async () => {
        const signaturesResult = await getSignaturesForLinking()
        const { parentWalletAddress, linkedWalletAddress, parentMessage, linkingMessage, parentSignature, linkedSignature } = signaturesResult
        const connectorName = connections[0]?.connector.name;

        const response = await api.linkWallet({
            signatureChainId: "137", // network chosen for both embedded wallet and Web SDK
            linkedWalletType: connectorName,
            parentWalletAddress,
            parentWalletMessage: parentMessage,
            parentWalletSignature: parentSignature,
            linkedWalletAddress: linkedWalletAddress!,
            linkedWalletMessage: linkingMessage,
            linkedWalletSignature: linkedSignature,
        })

        console.log(`status: ${response.status}`)
    }
}
```

## 3. Recuperando Wallets Vinculadas

Para proteger la privacidad de los usuarios, puedes pasar una firma generada desde la Embedded Wallet principal para recuperar la lista de direcciones de wallet que están vinculadas a una dirección de wallet principal.

La API devolverá los objetos de billetera vinculada en el siguiente formato:

```typescript
interface LinkedWallet {
  id: number
  walletType?: string
  walletAddress: string
  linkedWalletAddress: string
  createdAt?: string
}

interface GetLinkedWalletsReturn {
  linkedWallets: Array<LinkedWallet>
}
```

Y esto se puede lograr de la siguiente manera, pasando un mensaje, obteniendo una firma de la Embedded Wallet principal, verificando que la firma sea legible, y luego pasando las variables:`parentWalletAddress`,`parentWalletMessage`,`parentWalletSignature` y `signatureChainId` a la API:

```typescript
const getLinkedWallets = async () => {
    const parentWalletAddress = await sequenceWaas.getAddress()
    const message = "parent wallet with address " + parentWalletAddress
    const signature = await sequenceWaas.signMessage({
        message: message,
    })

    if (!signature.data.signature) {
        console.error("Could not get signature from wallet to be linked")
        throw new Error("Could not get signature from wallet to be linked")
    }

    const response = await api.getLinkedWallets({
        parentWalletAddress: parentWalletAddress as `0x${string}`,
        parentWalletMessage: message,
        parentWalletSignature: signature.data.signature,
        signatureChainId: "137",
    })

    response.linkedWallets.map((linkedWallet: LinkedWallet) => console.log(linkedWallet))

    return response.linkedWallets
}
```

## 4. Eliminación de Billeteras Vinculadas

Finalmente, si ya no deseas mantener una billetera vinculada fuera de la cadena a una billetera principal, puedes realizar una firma con ambas instancias de billetera (Embedded Wallet y billetera vinculada de Web SDK), y pasar la dirección de la billetera vinculada, [como en el paso 6](/solutions/wallets/link-wallets/integration-guide#6-link-sequence-kit-signed-in-wallet) para eliminar el estado de vinculación:

### Firmar Mensaje para Atestiguar la Eliminación de una Dirección Vinculada

```typescript
import {
  ...
  useAccount,
  useSignMessage
  ...
} from "wagmi";

function App() {

    const { address: kitWalletAddress } = useAccount();
    const { signMessageAsync } = useSignMessage();

    const getSignaturesForUnlinking = async () => {
        const parentWalletMessage = "linked wallet with address "
        const linkedWalletMessage = "parent wallet with address "
        const parentWalletAddress = await sequenceWaas.getAddress()
        const linkingMessage = "Unlink from " + linkedWalletMessage + parentWalletAddress

        const response: any = await signMessageAsync({
            message: linkingMessage,
        })

        const parentSignatureRes = await sequenceWaas.signMessage({
            message: parentMessage
        })

        const parentSignature = parentSignatureRes.data.signature
        const linkedSignature = response
        const linkedWalletAddress = kitWalletAddress

        return { parentWalletAddress, linkedWalletAddress, parentMessage, linkingMessage, parentSignature, linkedSignature }
    }
}
```

### Usar API para Agregar Carga Útil de Firma para la Dirección Principal

```typescript
function App(){
    ...
    const unLinkWallet = async () => {
        const signaturesResult = await getSignaturesForUnlinking()
        const { parentWalletAddress, linkedWalletAddress, parentMessage, linkingMessage, parentSignature, linkedSignature } = signaturesResult;

        const response = await api.removeLinkedWallet({
            signatureChainId: "137", // network chosen for both Embedded Wallet and Web SDK
            parentWalletAddress,
            parentWalletMessage: parentMessage,
            parentWalletSignature: parentSignature,
            linkedWalletAddress: linkedWalletAddress!,
            linkedWalletMessage: linkingMessage,
            linkedWalletSignature: linkedSignature,
        });

        console.log(`status: ${response.status}`)
    }
}
```
