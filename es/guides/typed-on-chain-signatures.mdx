## 1. Crear un Proyecto Builder y Obtener una Clave de Acceso

Primero, sigue [esta guía](/solutions/builder/getting-started) para crear un proyecto en Sequence Builder y obtener una clave de acceso del proyecto.

## 2. Inicializar la Aplicación React Vite

A continuación, comienza inicializando un nuevo proyecto que contendrá todo el código necesario para generar firmas y respuestas de validación desde la blockchain:

```shell
pnpm create vite
```

Esto debería crear un proyecto en blanco al que puedes empezar a añadir elementos y lógica.

## 3. Usar Sequence Wallet para el Inicio de Sesión del Usuario

Instala los paquetes necesarios requeridos para que el proyecto funcione:

```shell
pnpm install 0xsequence ethers
```

Luego habilita a un usuario para iniciar sesión con tu red elegida y la clave de acceso del proyecto obtenida en el [paso 1](/guides/typed-on-chain-signatures#1-create-a-builder-project--obtain-an-access-key).

```typescript

import { sequence } from '0xsequence'

function App() {
    sequence.initWallet(PROJECT_ACCESS_KEY, {
        defaultNetwork: 'sepolia',
    });

    const signIn = async () => {
        const wallet = sequence.getWallet()
        const details = await wallet.connect({app: 'sequence signature validation demo'})

        if(details){
            console.log('is signed in')
            console.log(details)
        }
    }

    return (
        ...
        <button onClick={() => signIn()}>sign in</button>
        ...
    )
}
```

## 4. Usar Datos Tipados EIP712 para Generar Firmas EIP6492

A continuación, definiremos datos tipados personalizados en typescript y usando la biblioteca de utilidades de Sequence construyendo un `TypedData` tipo, donde verificaremos una estructura de mensaje con `name`, `wallet`, y `message` parámetros:

En este ejemplo `VERIFYING_CONTRACT_ADDRESS` es el contrato inteligente que desplegamos en `sepolia` pero te mostraremos en el siguiente paso qué hace este contrato para que puedas desplegarlo tú mismo en cualquier red:

```typescript
import { sequence } from '0xsequence'

interface Person {
  name: string;
  wallet: string;
  message: string;
}

const VERIFYING_CONTRACT_ADDRESS = '0xB81efF8d6700b83B24AA69ABB18Ca8f9F7A356c5'
const CHAIN_ID = 11155111

const submitSignature = () => {
    const wallet = sequence.getWallet()

    const message = 'hey' // message can be dynamic
    const person: Person = {
        name: "user", // name can be dynamic
        wallet: wallet.getAddress(),
        message: message,
    };

    const chainId = CHAIN_ID
    const typedData: sequence.utils.TypedData = {
        domain: {
            // Domain settings must match verifying contract
            name: "Sequence Signature Validation Demo",
            version: "1",
            chainId,
            verifyingContract: VERIFYING_CONTRACT_ADDRESS,
        },
        types: {
            Person: [
                { name: "name", type: "string" },
                { name: "wallet", type: "address" },
                { name: "message", type: "string" },
            ],
        },
        message: person,
        primaryType: "Person",
    };
    ...
}

```

Luego firmaremos el objeto de mensaje tipado con las diversas propiedades referenciadas:

```typescript
const wallet = await sequence.getWallet()
const signer = wallet.getSigner(CHAIN_ID);

const signature = await signer.signTypedData(
    typedData.domain,
    typedData.types,
    typedData.message,
    {
        chainId,
        eip6492: true, // enabling signatures for non-deployed wallet contracts
    }
);

console.log("signature", signature);
```

Genial, adjunta la función a un botón y observa cómo se genera la firma después de que un usuario haya hecho clic en el botón:

```typescript
<button onClick={() => submitSignature()}>verify signature</button>
```

## 5. Desplegar Contrato para Verificación EIP712 y Validación EIP1271

Ahora proporcionaremos código fuente que puedes usar en algo como [Remix](https://remix.ethereum.org/) para desplegar un contrato, o incluso algo como [Foundry](https://book.getfoundry.sh/reference/forge/forge) para construir y desplegar con el [Sequence Builder](https://sequence.build)

<Steps>
  <Step title="Universal Signature Validator">
    El Validador Universal de Firmas puede hipotéticamente ser desplegado una vez para una red específica, y compartido con muchas aplicaciones, haciéndolo componible y reutilizable. Su uso es tanto para billeteras de contratos inteligentes fuera de la cadena como en la cadena para [EIP6492](https://eips.ethereum.org/EIPS/eip-6492) billeteras habilitadas.

    Puedes encontrar [el código fuente aquí](https://github.com/0xsequence-demos/EIP712-EIP1271-signatures-demo/blob/master/contracts/src/UniversalSignatureValidator.sol) que puedes usar para desplegar.
  </Step>

  <Step title="Custom Contract Verifier">
    El siguiente contrato lo explicaremos más en profundidad con las diversas funciones, ya que este contrato puede ser personalizado para la aplicación específica. Comienza con los siguientes conceptos básicos con el Validador Universal de Firmas pasado en el constructor en el primer paso:

    ```javascript
    import {IERC1271} from "./interfaces/IERC1271.sol";
    import {IERC6492} from "./interfaces/IERC6492.sol";

    import {ECDSA} from "@openzeppelin/contracts/utils/cryptography/ECDSA.sol";
    import {EIP712} from "@openzeppelin/contracts/utils/cryptography/EIP712.sol";

    struct Person { // can be customized
        string name;
        address wallet;
        string message;
    }

    contract EIP712Verifier is EIP712 {
        using ECDSA for bytes32;

        IERC6492 public immutable ERC6492_SIGNATURE_VALIDATOR; // the universal signature validator

        bytes32 private constant _ERC6492_DETECTION_SUFFIX = 0x6492649264926492649264926492649264926492649264926492649264926492;

        // this line of code must be customized to the struct you're verifying
        bytes32 private constant _PERSON_TYPEHASH = keccak256(bytes("Person(string name,address wallet,string message)"));

        constructor(address erc6492SignatureValidator) {
            ERC6492_SIGNATURE_VALIDATOR = IERC6492(erc6492SignatureValidator);
        }
        ...
    }

    ```
  </Step>

  <Step title="Verify Signature">
    A continuación, tenemos la función de verificación de firma que hace ambas cosas: crea el hash del mensaje y valida al firmante:

    ```javascript
    /// @dev Verifies the signature of a person.
    function verifySignature(address signer, Person memory person, bytes calldata signature)
        external
        returns (bool success)
    {
        bytes32 digest = personDigest(person);
        return ERC6492_SIGNATURE_VALIDATOR.isValidSig(signer, digest, signature);
    }
    ```

    #### Resumen Personalizado de Persona

    En la siguiente función recreamos el hash de la estructura con los parámetros pasados, que puede extenderse para incluir más o menos parámetros con tipos variados:

    Para más información sobre la construcción del resumen, consulta la [especificación EIP712](https://github.com/ethereum/EIPs/blob/master/EIPS/eip-712.md).

    ```javascript
    /// @dev Returns the EIP712 hash of a person.
    function personDigest(Person memory person) public view returns (bytes32 digest) {
        bytes32 structHash = keccak256(
            abi.encode(_PERSON_TYPEHASH, keccak256(bytes(person.name)), person.wallet, keccak256(bytes(person.message)))
        );
        digest = EIP712._hashTypedDataV4(structHash);
    }
    ```

    #### Validar Firmante

    A continuación, validamos el`signer` dirección, el`digest` y`signature`. Si se ha proporcionado una firma EIP6492, usamos el[validador universal de firmas](https://eips.ethereum.org/EIPS/eip-6492), de lo contrario verificamos la[EIP1271](https://eips.ethereum.org/EIPS/eip-1271) validación de firma directamente:

    ```javascript
    /// @dev Validates the ERC1271 signature of a signer.
    function validateSigner(address signer, bytes32 digest, bytes calldata signature) internal returns (bool success) {
        if (signature.length >= 32) {
            bool isCounterfactual =
                bytes32(signature[signature.length - 32:signature.length]) == _ERC6492_DETECTION_SUFFIX;
            if (isCounterfactual) {
                return ERC6492_SIGNATURE_VALIDATOR.isValidSig(signer, digest, signature);
            }
        }

        try IERC1271(signer).isValidSignature(digest, signature) returns (bytes4 magicValue) {
            return magicValue == IERC1271.isValidSignature.selector;
        } catch {}
        return false;
    }
    ```
  </Step>
</Steps>

Estás listo para desplegar ambos contratos, asegúrate de elegir tu red.

## 6. Renderizar Respuesta de Verificación y Validación del Contrato

Pasamos las firmas y llamamos al contrato desplegado usando`ethers` con el`PROJECT_ACCESS_KEY` en los siguientes pasos:

<Steps>
  <Step title="Create a Provider">
    Crea un proveedor usando la clave de acceso del proyecto:

    ```typescript
    import { ethers } from 'ethers'

    const CHAIN_HANDLE = 'sepolia'

    const provider = new ethers.JsonRpcProvider(
        `https://nodes.sequence.app/${CHAIN_HANDLE}/${PROJECT_ACCESS_KEY}`
    );
    ```
  </Step>

  <Step title="Initialize an Ethers Contract">
    Importa el ABI que fue generado del[paso 5](/guides/typed-on-chain-signatures#5-deploy-contract-for-eip712-verification-and-eip1271-validation) (o copia[del código fuente de git](https://github.com/0xsequence-demos/EIP712-EIP1271-signatures-demo/blob/master/src/abi.ts)), incluye el proveedor, e ingresa la dirección del contrato de verificación:

    ```typescript
    import { ABI } from "./abi";

    const contract = new ethers.Contract(
        VERIFYING_CONTRACT_ADDRESS,
        ABI,
        provider
    );
    ```
  </Step>

  <Step title="Static Call the Verify Signature Function">
    Al realizar una llamada estática a la función, simulamos la transacción sin enviarla a la cadena. Esto devuelve un resultado que especifica si la validación fue verdadera o falsa:

    ```typescript
    const address = await wallet.getAddress()

    const person: Person = {
        name: "user",
        wallet: address,
        message: message,
    }

    const signature = await signer.signTypedData(
        typedData.domain,
        typedData.types,
        typedData.message,
        {
            chainId,
            eip6492: true,
        }
    );

    const result = await contract.verifySignature.staticCall(
        address,
        person,
        signature
    );

    console.log(`Signature is ${result ? "valid" : "invalid"}`);
    return result;
    ```
  </Step>
</Steps>

## Conclusión

Ahora que tenemos estructuras de mensajes siendo enviadas a una blockchain y mensajes verificados con sus entradas, podemos extender la aplicación a una multitud de casos de uso que aseguran que los usuarios están firmando la información correcta (por ejemplo, permitir gastar ERC20's, realizar ofertas fuera de la cadena, código QR que contiene firma para acuñación aprobada, etc.).
