Tiempo para completar: 20-30 minutos

En esta guía, te mostraremos cómo usar la [Sequence Builder](https://sequence.build/) función de análisis para consultar información sobre el uso de tus usuarios para tu proyecto específico aprovechando un [Cloudflare Worker](https://www.cloudflare.com/).

Muestra a tu comunidad lo bien que lo estás haciendo a través de un [Dune](https://dune.com/) dashboard para dar una sensación de conectividad, o utiliza la API producida para incorporar bucles de retroalimentación inteligentes en tu juego impulsados por análisis de usuarios.

Puedes ver un ejemplo de la salida para esta guía [aquí](https://dune.com/mmhorizon/dungeon-minter-analytics)

1. Gestión de Claves de Acceso: Reclama una clave de acceso secreta para interactuar con el stack de Sequence
2. Cloudflare Worker: Crea una función que consulta el stack de Sequence y produce puntos de datos específicos del proyecto
3. Dashboard de Dune: Crea una vista de los datos como un dashboard compartible

<Note>
  Una referencia al código de plantilla se puede encontrar [aquí](https://github.com/0xsequence-demos/template-cloudflare-worker-wallets-analytics)
</Note>

## 1. Gestión de Claves de Acceso

Necesitarás adquirir una Clave de Acceso Secreta para tu proyecto para autenticar tu aplicación con el stack de Sequence. Sigue estos pasos:

### Creación de Clave de Acceso Secreta

<Steps>
  <Step title="Access Settings">
    Primero comienza accediendo a la configuración y seleccionando la tarjeta de Claves API:

    <Frame>
      ![builder settings access keys](/images/builder/builder_settings_access_keys.png)
    </Frame>
  </Step>

  <Step title="Add Service Account">
    Desplázate hacia abajo y selecciona `+ Add Service Account`:

    <Frame>
      ![builder settings add service account](/images/builder/builder_settings_add_service_account.png)
    </Frame>
  </Step>

  <Step title="Select Write Permission">
    Luego cambia el permiso a `Write`, haz clic en `+ Add Service Account`, y selecciona `Confirm`:

    <Frame>
      ![builder settings add service account](/images/builder/builder_settings_add_service_account_confirm.png)
    </Frame>

    Finalmente `copy` la clave y guárdala en un lugar seguro, ya que no tendrás acceso a ella en el futuro desde el Sequence Builder.
  </Step>
</Steps>

## 2. Cloudflare Worker

En este ejemplo, utilizamos un Cloudflare Worker para experimentar el escalado automático basado en el uso del dashboard y despliegues fáciles desde la cli, pero por supuesto puedes usar tu propio backend u otras alternativas serverless.

<Steps>
  <Step title="Create Project">
    Para crear el proyecto desde cero, primero crea un proyecto con `mkdir`, `cd` entra al proyecto, y ejecuta `pnpm init` para crear un `package.json`.
  </Step>

  <Step title="'Hello World' Worker">
    Asegúrate de que wrangler cli esté instalado en tu proyecto y establece la palabra clave `wrangler` como un alias en tu sesión local de bash.

    ```shell
    pnpm install wrangler --save-dev
    alias wrangler='./node_modules/.bin/wrangler'
    ```

    Crea una cuenta en el [sitio de Cloudflare](https://cloudflare.com/) y realiza un paso de inicio de sesión para acceder a tu dashboard de Cloudflare para conectar la plataforma Cloudflare a tu entorno de desarrollo local.

    ```shell
    wrangler login
    ```

    Una vez iniciada la sesión, inicializa el proyecto en el directorio con el comando `wrangler init` y acepta uno de los nombres de carpeta de proyecto generados aleatoriamente que te guste, y sigue las indicaciones para inicializar tu aplicación `"Hello World" Worker` con control de versiones git.

    ```shell
    wrangler init
    ```

    Para completar este paso, deberías presionar enter 4 veces después de `wrangler init` con los últimos 2 pasos respondidos como `No` para rechazar el control de versiones git y el despliegue.

    Esto clonará un repositorio inicial que puede ser usado para desplegar código en la nube.

    <Note>
      Pruebas de API Local <br />
      En cualquier momento de la guía, puedes usar el comando `wrangler dev` en la carpeta del proyecto para pruebas locales
    </Note>

    #### Prueba de Despliegue

    Finalmente, `cd` entra en la carpeta del proyecto generada aleatoriamente y ejecuta el comando `wrangler deploy`.

    Esto debería imprimir una URL, que puedes ingresar en el navegador `https://<app>.<account>.workers.dev` para ver el `Hello World!` resultado.
  </Step>

  <Step title="Setup Config, Routes, & Mock functions">
    Una vez que tengas el proyecto configurado, actualiza tu `wrangler.toml` con las siguientes variables, donde `DAYS` es el período de tiempo que estás observando:

    ```shell
    [vars]
    SECRET_API_ACCESS_KEY = "<SECRET_API_ACCESS_KEY>"
    PROJECT_ID = <PROJECT_ID>
    DAYS = <DAYS>
    ```

    Luego incluye el tipo `Env` con las variables en `index.ts`:

    ```ts
    export interface Env {
    	PROJECT_ID: number;
    	SECRET_API_ACCESS_KEY: string;
    	DAYS: number;
    }
    ```

    Reemplaza la función `fetch` existente con las siguientes llamadas de función simuladas:

    ```ts
    export default {
    	async fetch(request: Request, env: Env, ctx: ExecutionContext): Promise<Response> {
    		const url = new URL(request.url);
    		
    		// Handle different endpoints
    		if (url.pathname === "/dailyActiveUsers") {
    			return handleDailyWallets(env, request);
    		} else if (url.pathname === "/totalTransactionsSent") {
    			return handleTotalTxns(env, request);
    		} else {
    			return new Response("No function for this URL", { status: 405 });
    		}
    	},
    };
    ```

    Usando las siguientes funciones:

    ```ts
    const handleDailyWallets = async (env: Env, request: Request) => {
    	return new Response(JSON.stringify({endpoint: 'daily'}), { status: 200 });
    }

    const handleTotalTxns = async (env: Env, request: Request) => {
    	return new Response(JSON.stringify({endpoint: 'total'}), { status: 200 });
    }
    ```
  </Step>

  <Step title="Date Formatting">
    A continuación, incluye las siguientes funciones de utilidad para analizar la fecha correcta del valor actualizado en el `wrangler.toml` para la `DAYS` variable:

    ```ts
    const endDate = () => {
    	const today = new Date();
    	return today.toISOString().substring(0, 10); // only including the YYYY-MM-DD date
    }

    const startDate = (env: Env) => {
    	const today = new Date();

    	// Format today's date as a string
    	const daysBefore = new Date(today);
    	daysBefore.setDate(daysBefore.getDate() - env.DAYS);
    	
    	// Format the date 7 days before as a string by only including the YYYY-MM-DD date
    	const daysBeforeString = daysBefore.toISOString().substring(0, 10);
    	return daysBeforeString
    }
    ```
  </Step>

  <Step title="Daily Active Users">
    Ahora, maneja la solicitud `Daily Active Users` usando la siguiente función, que llama a la API de Analytics de Sequence:

    ```ts
    const handleDailyWallets = async (env: Env, request: Request) => {
    	const resp = await fetch(
    	`https://api.sequence.build/rpc/Builder/WalletsDaily`,
    		{
    			method: 'POST',
    			headers: {
    			'Content-Type': 'application/json',
    			'Authorization': `Bearer ${env.SECRET_API_ACCESS_KEY}`
    			},
    			body: JSON.stringify({
    			filter: {
    				dateInterval: 'DAY',
    				endDate: endDate(),
    				projectId: env.PROJECT_ID,
    				startDate: startDate(env)
    			}
    			})
    		}
    	)
    	
    	const data: any = await resp.json();
    	return new Response(JSON.stringify(data.walletStats), { status: 200 });
    }
    ```
  </Step>

  <Step title="Total Transactions Sent">
    Finalmente, agrega la siguiente función para el `Total Transactions Sent`:

    ```ts
    const handleTotalTxns = async (env: Env, request: Request) => {
    	const resp = await fetch(
    	`https://api.sequence.build/rpc/Builder/WalletsTxnSentTotal`,
    		{
    		  method: 'POST',
    		  headers: {
    			'Content-Type': 'application/json',
    			'Authorization': `Bearer ${env.SECRET_API_ACCESS_KEY}`
    		  },
    		  body: JSON.stringify({
    			filter: {
    			  dateInterval: 'DAY',
    			  endDate: endDate(),
    			  projectId: env.PROJECT_ID,
    			  startDate: startDate(env)
    			}
    		  })
    		}
    	)
    	
    	const data: any = await resp.json();
    	return new Response(JSON.stringify(data.walletStats), { status: 200 });
    }
    ```
  </Step>

  <Step title="Included Spacing for Days of Zero Data">
    Incluido en la API de Analytics de Sequence están los días con actividad cero eliminados de la respuesta. Sin embargo, si deseas incluir días con datos cero en tus consultas de Dune para mostrar el espaciado del tiempo en relación con los datos, puedes usar la siguiente función para completar los días que no muestran datos en el formato de fecha correcto:

    ```typescript
    const fillMissingDates = (data: any[], startDate: string, endDate: string) => {
    	const filledData: { value: number, label: string }[] = [];
    	const start = new Date(startDate);
    	const end = new Date(endDate);
    	
    	for (let d = new Date(start); d <= end; d.setDate(d.getDate() + 1)) {
    		const dateString = d.toISOString().substring(0, 10);
    		const existingData = data.find(entry => entry.label === dateString);
    		if (existingData) {
    			filledData.push(existingData);
    		} else {
    			filledData.push({ value: 0, label: dateString });
    		}
    	}
    	
    	return filledData;
    }
    ```

    Luego para **ambas** respuestas de llamada incluye el siguiente método con los datos `walletStats` pasados:

    ```typescript
    	...
    	const data: any = await resp.json();
    	const filledData = fillMissingDates(data.walletStats, startDate(env), endDate());
    	return new Response(JSON.stringify(filledData), { status: 200 });
    }
    ```
  </Step>
</Steps>

Ahora puedes probar tu API llamando a las rutas respectivas con el nombre de host (`/dailyActiveUsers` y `/totalTransactionsSent`) una vez que hayas redesplegado con `wrangler deploy`.

<Note>
  Para más ejemplos de endpoints disponibles a través de la API de Analytics, por favor consulta la [descripción general](/api-references/analytics/overview) página.
</Note>

## 3. Panel de Dune

<Steps>
  <Step title="Dune Sign Up">
    Primero, regístrate en [Dune](https://dune.com/)
  </Step>

  <Step title="Create Query">
    Accede a tu cuenta en `https://dune.com/<account>` y selecciona el `Create` botón y `New query`.

    <Frame>
      ![dune create query](/images/guides/analytics/dune_create_query.png)
    </Frame>
  </Step>

  <Step title="Daily Active Users Query">
    Ingresa la siguiente consulta SQL en la consola y selecciona `Run`:

    ```sql
    SELECT
      t.label as "Date", -- converting the label to "Date"
      t.value as "Count" -- converting the value field to "Count"
    FROM UNNEST(
      TRY_CAST(
        JSON_PARSE(HTTP_GET('https://<URL>/dailyActiveUsers')) AS ARRAY(ROW(label VARCHAR, value DOUBLE))
      )
    ) AS t(label, value)

    ```

    Una vez que se hayan devuelto los resultados, crea un `New visualization`.

    Luego, selecciona `Add visualization` después del `Bar chart` predeterminado seleccionado del menú desplegable (pero siéntete libre de personalizarlo).

    <Frame>
      ![add visualization](/images/guides/analytics/dune_add_visualization.png)
    </Frame>

    Finalmente, haz clic en `Save` y agrega un nombre a tu consulta.

    <Frame>
      ![save query](/images/guides/analytics/dune_query_save.png)
    </Frame>
  </Step>

  <Step title="Total Transactions Sent Query">
    Repite los pasos del [paso anterior](/guides/analytics-guide#create-query) y usa la siguiente consulta SQL:

    ```sql
    SELECT
      t.label,
      t.value
    FROM UNNEST(
      TRY_CAST(
        JSON_PARSE(HTTP_GET('https://<URL>/totalTransactionsSent')) AS ARRAY(ROW(label VARCHAR, value DOUBLE))
      )
    ) AS t(label, value)

    ```

    Una vez que se hayan devuelto los resultados, crea un `New visualization`.

    Luego, selecciona `Add visualization` y desplázate hasta `Counter` para crear un widget contador que devuelva el total absoluto retornado por la API.
  </Step>

  <Step title="Create New Dashboard">
    Accede a los botones que dicen `Create` > `New dashboard` e ingresa un nombre para el nuevo panel.

    <Frame>
      ![dune create dashboard](/images/guides/analytics/dune_create_dashboard.png)
    </Frame>

    Una vez creado, agrega las 2 consultas anteriores seleccionando `Edit` y `Add visualization`.

    <Frame>
      ![dune edit dashboard](/images/guides/analytics/dune_edit_dashboard.png)
    </Frame>

    <Frame>
      ![dune add visualization from dashboard](/images/guides/analytics/dune_add_visualization_from_dashboard.png)
    </Frame>

    Una vez por cada consulta, busca el nombre en el modal y selecciona `Add` para cada consulta, luego `Done` en el modal, y `Done` en el panel.
  </Step>
</Steps>
