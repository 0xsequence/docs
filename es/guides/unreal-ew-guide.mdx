# Introducción a Cropout - Guía de Juego en Unreal Engine

Cropout es un juego RTS casual con vista superior construido con Unreal Engine, en esta guía te mostraremos cómo integrar las funciones de Embedded Wallet usando el SDK de Sequence para Unreal.

Con esta integración, los jugadores podrán acceder a funciones como billeteras inteligentes, propiedad de activos en el juego y gestión de transacciones, todo sin ventanas emergentes disruptivas o requisitos de tarifas de gas.

Revisa el [Boilerplate para esta guía.](https://github.com/0xsequence/sequence-unreal)

[Aprende más sobre Embedded Wallets.](/solutions/wallets/overview#what-are-the-differences-between-these-options)

## 1. Configuración

### Descarga el ejemplo del juego

Si tienes Unreal Engine instalado ve directamente a [la página de Cropout en Unreal](https://www.unrealengine.com/en-US/blog/cropout-casual-rts-game-sample-project).

Si no, comienza instalando Epic Games Launcher:

1. Descarga e instala el launcher desde [el sitio web de Epic Games](https://www.unrealengine.com/en-US/download).

2. Iniciar Sesión: Inicia sesión o crea una cuenta de Epic Games.

3. Encuentra el Proyecto: Ve a la pestaña de Unreal Engine, luego a la sección Learn, y busca "Cropout Sample Project".

4. Descargar: Haz clic en Agregar a la Biblioteca y luego en Instalar para descargar los archivos del proyecto.

## Descarga el SDK de Sequence para Unreal Engine

Para comenzar, descarga la [última versión](https://github.com/0xsequence/sequence-unreal/releases) del SDK de Sequence y coloca la `SequencePlugin` carpeta en tu proyecto de Unreal `Plugin` carpeta.

Alternativamente, [obtén el SDK desde el Epic Games Marketplace.](/sdk/unreal/installation#using-epic-games-marketplace)

### Configura tu Proyecto

1. [Inicia sesión en la Consola de Sequence Builder.](/solutions/builder/getting-started)

2. [Crea un Proyecto Para Tu Juego en la Consola de Builder.](/support/builder/project-management)

3. Configura una [Embedded Wallet en la Consola de Builder.](/solutions/builder/embedded-wallet#embedded-wallet-in-builder)

4. En tu `SequenceConfig.ini` archivo, agrega tu `WaaS Config Key` y tu Clave API de Builder del [Consola del Constructor](/solutions/builder/getting-started#claim-an-api-access-key) bajo `Settings > API Access Keys`.

5. No olvides agregar tus ids de cliente Oauth preferidos, que debes configurar en el Constructor bajo `Onboard > Embedded Wallet > Configuration`.

### Iniciar sesión usando la interfaz incorporada

En el plano del Menú Principal, generaremos nuestro `Sequence Connector` Actor que iniciará y mostrará el Panel de Inicio de Sesión.

Recomendamos usar la interfaz incorporada para propósitos de desarrollo, aquí hay una [explicación más detallada. ](/sdk/unreal/user_interfaces/)

Para mayor personalización, [aprende cómo crear tu propia interfaz.](/sdk/unreal/authentication/)

## 2. Configurando el Id de Red y la Cadena

Para comenzar nuestra interacción con el SDK de Sequence, abre el *Sequence Connector* blueprint, localiza el `Auth Success` evento en *Event Graph > Begin Play*, y llamaremos `Api Update Network Id` desde el Subsistema SequenceWalletBP para configurar la billetera en la cadena seleccionada de tu proyecto constructor.
En el mismo evento, también llamaremos `Set Chain` desde el Subsistema Sequence Indexer para que podamos obtener correctamente los activos.

## 3. Obtener dirección de la billetera

Para mostrar la dirección de la billetera, primero recuperamos la referencia de la billetera del usuario nuevamente usando SequenceWalletBP y la usamos para obtener y mostrar la dirección de la billetera de la red seleccionada usando `Api GetWallet Address` desde el Subsistema SequenceWalletBP. Mostraremos la dirección del usuario en UI\_Layer\_Menu.

## 4. Configura tus NFTs

En nuestro caso, estamos comenzando con un contrato ERC-1155, para nuestros paquetes de Resource Bundle y Farming Boosts. Para crear los NFTs ve a [Builder Console >  Deploy > Contracts](/solutions/builder/contracts/) y selecciona `Web3 Game Item`.

Una vez que hayamos creado nuestros elementos, ahora crearemos sus respectivos contratos de venta desplegando un nuevo contrato y seleccionando `Web 3 Game Item(Sale)`.

Agregaremos a nuestros elementos del Juego Web 3 sus recompensas correspondientes y la dirección del contrato de venta como propiedad.

### Obteniendo nuestros NFTs

Para recuperar nuestros NFTs recién creados, usaremos el subsistema Indexer para llamar a Get Token Supply, que nos devolverá la información y metadatos para la dirección del contrato dada. Encontrarás esto en el blueprint `CUI_Purchase`.

### Comprando un NFT

Para permitir que el jugador compre un NFT, usaremos el subsistema SequenceSDK para crear y enviar una `PrimaryPurchase` transacción del deseado `Web 3 Game Item(Sale)` y usaremos la propiedad "Sale" de los contratos para referenciar nuestro `Web 3 Game Item(Sale)`. Encontrarás esto en el blueprint `CUI_PurchaseItem`.

### Obtener los NFTs del jugador

Para recuperar los NFTs del jugador usaremos el Subsistema Indexer para llamar a `Get Token Balances`, que nos devolverá todos los Balances de contrato para la dirección de billetera dada. Una vez devuelto, llamaremos a Get Token Balances nuevamente, pero esta vez para cada contrato específico, lo que nos devolverá el balance más la información y metadatos para el contrato dado. Encontrarás esto en el blueprint `CUI_Vault`.

### Quemar los NFTs del jugador

Para dar al jugador las recompensas por usar uno de nuestros NFTs, usaremos el subsistema SequenceSDK para crear y enviar una `Burn` transacción del deseado `Web 3 Game Item` que nos devolverá el estado de la transacción y la dirección del contrato, para verificar el éxito de la quema de elementos. En caso de éxito, usaremos las propiedades del contrato para dar a los jugadores su recompensa. Encontrarás esto en el blueprint `CUI_VaultItem`.

Para una explicación más detallada, ve a [Leer de la Blockchain](/sdk/unreal/read-from-blockchain/) y [Escribir en la Blockchain](/sdk/unreal/write-to-blockchain/) secciones.

## 5. Cerrar sesión del jugador

Para cerrar la sesión del jugador y terminar su sesión, llamaremos a \`Clear Session\` desde el Subsistema Sequence Sessions, de esta manera eliminaremos sus credenciales almacenadas y el usuario necesitará iniciar sesión nuevamente cuando regrese. Llamaremos a este método en UI\_Layer\_Menu.
