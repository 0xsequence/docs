---
title: Email + OTP
---

Disponible en todas las plataformas, el inicio de sesión por correo electrónico proporciona al usuario un desafío de Contraseña de Un Solo Uso (OTP) - un código de 6 dígitos enviado por correo electrónico a la dirección ingresada para que el usuario lo introduzca en la siguiente página.

Primero, querrás [habilitar el inicio de sesión por correo electrónico para tu proyecto en el builder](/solutions/builder/embedded-wallet/configuration).

Si estás usando el `LoginPanel`, la interfaz de usuario gestionará automáticamente el inicio de sesión por correo electrónico + OTP.

<Tip>¡No olvides suscribirte al `SequenceWallet.OnWalletCreated` evento para recibir tu billetera recién creada!</Tip>

## Integraciones Personalizadas

De lo contrario, querrás llamar al `Login(string email)` Task asíncrono en `SequenceLogin` para iniciar la autenticación con la API; esto enviará el desafío OTP al correo electrónico especificado.

Una vez que el usuario haya introducido su código OTP, querrás llamar al `Login(string email, string code)` Task asíncrono en `SequenceLogin` para finalizar la autenticación del usuario.

Los eventos relevantes incluyen:

* `OnMFAEmailSent` - este evento incluye una cadena que especifica el correo electrónico donde se envió el OTP

* `OnMFAEmailFailedToSend` - este evento incluye dos cadenas: el correo electrónico donde se solicitó el OTP y el mensaje de error

* `OnLoginFailed` - este evento incluye: el mensaje de error, el `LoginMethod` que se utilizó, y la cadena de correo electrónico asociada con el intento de inicio de sesión
