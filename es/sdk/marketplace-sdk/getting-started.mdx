Antes de comenzar esta integración, asegúrate de haber instalado y configurado nuestro Web SDK. Para instrucciones de configuración, por favor consulta la [Web SDK - Comenzando](/sdk/web/getting-started) documentación. Una vez completado, regresa aquí para continuar.

De lo contrario, te guiaremos a través del proceso de instalación de Marketplace SDK, inicialización del SDK y te proporcionaremos algunos hooks para trabajar con nuestro marketplace

## Instalando los Paquetes de Marketplace SDK

Marketplace SDK es modular, permitiéndote instalar solo los paquetes necesarios. Para comenzar, instala el `@0xsequence/marketplace-sdk` paquete principal, así como instalar otras dependencias necesarias.

```bash
npm install @0xsequence/kit @0xsequence/kit-checkout @0xsequence/kit-wallet @0xsequence/marketplace-sdk @0xsequence/design-system@^1 @0xsequence/network wagmi ethers@^6 viem 0xsequence @tanstack/react-query @tanstack/react-query-devtools @legendapp/state framer-motion@^8.5.2 pino-pretty
# or
pnpm add @0xsequence/kit @0xsequence/kit-checkout @0xsequence/kit-wallet @0xsequence/marketplace-sdk @0xsequence/design-system@^1 @0xsequence/network wagmi ethers@^6 viem 0xsequence @tanstack/react-query @tanstack/react-query-devtools @legendapp/state framer-motion@^8.5.2 pino-pretty
# or
yarn add @0xsequence/kit @0xsequence/kit-checkout @0xsequence/kit-wallet @0xsequence/marketplace-sdk @0xsequence/design-system@^1 @0xsequence/network wagmi ethers@^6 viem 0xsequence @tanstack/react-query @tanstack/react-query-devtools @legendapp/state framer-motion@^8.5.2 pino-pretty
```

# Configurando tu Dapp

Para utilizar el wrapper principal de marketplace-sdk para interactuar con tu marketplace desde tu aplicación, sigue estos pasos:

<Steps>
  <Step title="Verify If Your Web SDK Is Correctly Integrated">
    Asegurarse de que tu Web SDK esté correctamente integrado es crucial antes de comenzar con Marketplace SDK. Para verificar esto, simplemente comprueba si puedes iniciar y cerrar sesión exitosamente.
  </Step>

  <Step title="Create a Config">
    A continuación, será necesario crear una variable de configuración para Marketplace SDK. En este archivo, definiremos y gestionaremos la configuración esencial para nuestro Marketplace SDK, incluyendo variables de entorno. Este enfoque centralizado asegura que los valores de configuración sean fácilmente accesibles y mantenibles.

    #### config.ts

    <CodeGroup>
      ```ts vite
      import type { SdkConfig } from '@0xsequence/marketplace-sdk';

      const projectAccessKey = import.meta.env.VITE_PROJECT_ACCESS_KEY;
      const projectId = import.meta.env.VITE_PROJECT_ID!;
      const waasConfigKey = import.meta.env.VITE_WAAS_CONFIG_KEY;
      const googleClientId = import.meta.env.VITE_GOOGLE_CLIENT_ID;
      const appleClientId = import.meta.env.VITE_APPLE_CLIENT_ID;
      const appleRedirectURI = window.location.origin + window.location.pathname;
      const walletConnectId = import.meta.env.VITE_WALLET_CONNECT_ID;

      export function getConfig() {
        const embedded = waasConfigKey
          ? ({
              waasConfigKey,
              googleClientId,
              appleClientId,
              appleRedirectURI,
            } satisfies NonNullable<SdkConfig["wallet"]>["embedded"])
          : undefined;

        const config = {
          projectId,
          projectAccessKey,
          wallet: {
            walletConnectProjectId: walletConnectId,
            embedded,
          },
        } satisfies SdkConfig;

        return config;
      }
      ```

      ```ts nextjs server side rendering

            walletConnectProjectId: walletConnectId,

      ```

  <Step title="Create an SSR Client">
    A continuación, será necesario crear un Cliente Ssr para Marketplace SDK. Este Cliente SSR sirve como punto de entrada para inicializar el marketplace SDK en el servidor Next.js, permitiendo una eficiente obtención de datos y configuración antes de renderizar la UI.

    #### Entendiendo el Cliente SSR

    El Cliente SSR te permite acceder a datos y configuraciones clave del marketplace, que son esenciales para inicializar correctamente el SDK en el lado del servidor. Se pueden recuperar los siguientes datos:

    Esta configuración asegura una integración perfecta del SDK del marketplace con el renderizado del lado del servidor, mejorando el rendimiento y la experiencia del usuario.

    #### ssrClient.ts

    <CodeGroup>
      ```ts vite
      // You can omit this step with VITE
      ```

      ```ts nextjs server side rendering
      import { getConfig } from './config';
      import { createSSRClient } from '@0xsequence/marketplace-sdk/react/ssr';
      import { QueryClient } from '@tanstack/react-query';
      import { headers } from 'next/headers';

      export const ssrClient = async () => {
        const headersList = headers();
        
        return createSSRClient({
          cookie: headersList.get('cookie') || '',
          config: getConfig(`https://${headersList.get('x-forwarded-host') || headersList.get('host')}/`),
          queryClient: new QueryClient(),
        });
      };
      ```

      ```ts nextjs client side rendering
      // You can omit this step in NEXT.js if your layout uses 'use client' instead of server-side rendering
      ```
    </CodeGroup>
  </Step>

  <Step title="Add the Marketplace SDK Providers Alongside Your Web SDK Providers">
    Abre tu`Providers.tsx` archivo, donde está configurado el Web SDK, e integra los proveedores del SDK del Marketplace.

    <CodeGroup>
      ```ts vite
      import {
        MarketplaceProvider,
        ModalProvider,
      } from "@0xsequence/marketplace-sdk/react";
      import { getConfig } from "./config";

      const sdkConfig = getConfig();

      // Into your React component:

      return (
        /* Your other providers should be placed here (they should wrap MarketplaceProvider) */
        
        <MarketplaceProvider config={sdkConfig}>
          {children}
          <ModalProvider />
        </MarketplaceProvider>

        /* Your other providers should close here */
      );
      ```

      ```ts nextjs server side rendering
      import {
        MarketplaceProvider,

      import {
      ```
