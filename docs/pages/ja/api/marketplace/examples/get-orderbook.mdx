### MOCKDATA + ja-translated Schema
ja-translated Query the protocol for orders based on various filters.

MOCKDATA + ja-translated _Sequence Orderbook `GetOrderbookOrders` Method:_
- ja-translated Request: POST /rpc/Marketplace/GetOrderbookOrders
- ja-translated Content-Type: application/json
- MOCKDATA + ja-translated Body (in JSON):
  - MOCKDATA + ja-translated `orderbookContractAddress` (string) -- the address of the protocol contract
  - ja-translated `collectionAddress` (string) -- the address of the NFT collection
  - MOCKDATA + ja-translated `currencyAddresses` (string\[]) -- an array of currency contract addresses
  - MOCKDATA + ja-translated `filters` (OrderbookOrderFilter) -- filters to apply to the query
    - ja-translated `tokenIds` (string\[]) -- the token ids of the assets you're looking at
    - MOCKDATA + ja-translated `isListing?` (bool) -- `true` if it is an order, `false` if it's an offer
    - ja-translated `userAddress?` (string) -- the address of the user
  - MOCKDATA + ja-translated `orderStatuses` (OrderStatus\[]) (optional) -- filter orders by their statuses as `OPEN`, `CLOSED`, `CANCELLED` strings
  - ja-translated `metadataFilter` (Filter) (optional) -- filter orders by metadata
    - MOCKDATA + ja-translated `text?` (string) -- text to search
    - ja-translated `properties?` (PropertyFilter\[]) --
      - MOCKDATA + ja-translated `name` (string) -- name of the property, eg: hero
      - ja-translated `type`: (PropertyType) -- type of the values of the property as an enum from: `"INT"`,`"STRING"`,`"ARRAY"`, or `"GENERIC"`
      - MOCKDATA + ja-translated `min?`: (int64) -- used if type is int, gives range of values for filtering
      - ja-translated `max?`: (int64) -- used if type is int, gives range of values for filtering
      - MOCKDATA + ja-translated `values?`: \[]any -- if string then array of values, eg: \["Ada", "Ari", "Axel", "Banjo" ...]
  - ja-translated `beforeExpiryTimestamp` (int64) -- filter orders expiring before the given timestamp
  - MOCKDATA + ja-translated `page?` -- page object to sort by
    - ja-translated `sort` (Sort\[]) -- with an object of keys `"column"` and `"order"` with example `"createdAt"` and `"ASC"`, respectively
- ja-translated Response (in JSON):
  - MOCKDATA + ja-translated `page`
    - ja-translated `page` (i32) -- number of the page
    - MOCKDATA + ja-translated `pageSize` (i32) -- number of orders on the page
    - ja-translated `more` (bool) -- if there are more orders
  - MOCKDATA + ja-translated `orders` (OrderbookOrder\[])
    \[
    - ja-translated `orderId` (string) -- order id
    - MOCKDATA + ja-translated `tokenContract` (string) -- the collection contract
    - ja-translated `tokenId` (string) -- the tokenId of the collection in the order
    - MOCKDATA + ja-translated `isListing` (bool) -- is true if it's a listing for sale, or false if it's an offer
    - ja-translated `quantity` (string) -- the number of tokens the order is for
    - MOCKDATA + ja-translated `quantityRemaining` (string) -- the number of tokens after the order
    - ja-translated `currencyAddress` (string) -- the currency the token is trading in
    - MOCKDATA + ja-translated `pricePerToken` (string) -- the price per token
    - ja-translated `expiry` (string) -- the expiry of when the order is no longer valid
    - MOCKDATA + ja-translated `orderStatus` (OrderStatus) -- the status of the order as an enum fom `"OPEN"`, `"CLOSED"`, `"CANCELLED"`
    - ja-translated `createdBy` (string) -- the address the order was created by
    - MOCKDATA + ja-translated `createdAt` (uint64) -- the time the order was created
    - ja-translated `orderbookContractAddress` (string) -- the orderbook contract address
