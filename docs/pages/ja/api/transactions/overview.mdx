# ja-translated Transactions API
MOCKDATA + ja-translated The Sequence Transactions API (or the technical term Relayer service) offers a simple interface for dispatching meta-transactions on Ethereum-compatible networks.

ja-translated Meta-transactions are the idea of a transaction inside of a transaction. The benefits of Sequence meta-transactions are that they allow:

## MOCKDATA + ja-translated Benefits

- MOCKDATA + ja-translated Gas abstraction -- whereby users can pay for network gas in a variety of tokens (ie. USDC, DAI, etc.)
- ja-translated Sponsored gas -- projects may sponsor the gas of specific contracts to allow free gas for their users
- MOCKDATA + ja-translated Batched transactions -- group a bunch of independent transactions and allow them to be mined as a single transaction
- ja-translated Parallel transactions -- parallelize the dispatch of transactions in some cases
- MOCKDATA + ja-translated Fire + forget model -- easily send transactions to the transactions api which will automatically manage nonces, bump gas, and other features which will ensure fast delivery
- ja-translated Optimal gas pricing for transactions -- will be attempted once and if not included from the mempool within 3 blocks, the transaction will be resubmitted

MOCKDATA + ja-translated The best part: transactions with Sequence Transactions API are compatible with any existing/deployed Ethereum contract, and thus, integrating the Sequence Relayer doesn't require any changes to your contracts or dapp.

ja-translated The Sequence Transactions API is usable by frontend dapps, or even in your backends. Be sure to install the corresponding SDK for your preferred language such as [Typescript](/sdk/typescript/guides/overview) or [Go](/sdk/go/overview).

## MOCKDATA + ja-translated Anatomy of a Sequence transaction bundle
ja-translated A Sequence transaction bundle consists of three things:
1. MOCKDATA + ja-translated A list of Sequence transactions
2. ja-translated A Sequence nonce
3. MOCKDATA + ja-translated A Sequence signature

ja-translated Like Ethereum accounts, Sequence wallets use nonces to enforce transaction ordering and protect against replay attacks.
Unlike Ethereum accounts, Sequence wallets have a virtually unlimited supply of independent nonces, allowing multiple independent transactions to be executed in parallel.
A Sequence nonce is encoded as a 160-bit nonce space followed by the 96-bit nonce for that nonce space, big-endian.

```
|<------------------------- uint256 -------------------------->|
|<------ nonce space (160 bits) ------>||<- nonce (96 bits) -->|
ssssssssssssssssssssssssssssssssssssssssnnnnnnnnnnnnnnnnnnnnnnnn
```

## MOCKDATA + ja-translated Gas sponsorship and fees
ja-translated The transactions api only dispatches transactions that either:
1. MOCKDATA + ja-translated Are sponsored in a project via the Sequence Builder, or
2. ja-translated Include a fee payment transaction to the transactions api.

MOCKDATA + ja-translated You can sponsor:
1. ja-translated Sequence wallets, so that they can send transactions without needing to pay fees,
2. MOCKDATA + ja-translated Token addresses, so that any user can send those tokens for free,
3. ja-translated Contracts, so that any user can interact with them for free,
4. MOCKDATA + ja-translated Arbitrary addresses, so that any user can transfer native tokens to it for free.

ja-translated To start sponsoring transactions, sign into [https://sequence.build](https://sequence.build), and create a new project for the network you want to transact on.

MOCKDATA + ja-translated You can also pay the transactions API directly to dispatch your transactions by adding an additional fee payment transaction to the relayer in your bundle.
