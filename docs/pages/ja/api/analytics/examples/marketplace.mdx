# MOCKDATA + ja-translated Marketplace Analytics Examples
ja-translated Additionally, you can get deep insights into how your Sequence Marketplace is performing so you can report, track, and refine to earn more revenue.

ja-translated :::info
Replace the PROJECT\_ID and SECRET\_API\_ACCESS\_KEY variables with your project ID and secret token from Sequence Builder.
:::

## MOCKDATA + ja-translated Fetch Transactions on your Marketplace
ja-translated Fetching the number of transaction events on the Sequence marketplace - these can used either be by total or a fixed time interval.

### MOCKDATA + ja-translated Total
MOCKDATA + ja-translated :::code-group

```sh [cURL]
curl 'https://api.sequence.build/rpc/Builder/MarketTxnEventTotal' \
  -H 'accept: */*' \
  -H 'authorization: BEARER <SECRET_API_ACCESS_KEY>' \
  -H 'content-type: application/json' \
  --data-raw '{"filter":{"projectId":<PROJECT_ID>,"startDate":"2024-04-23","endDate":"2024-05-23"}}'
```

```ts twoslash [TypeScript]
// Works in both a Webapp (browser) or Node.js by importing cross-fetch:
// import fetch from "cross-fetch";
(async () => {
  const res = await fetch(
    "https://api.sequence.build/rpc/Builder/MarketTxnEventTotal",
    {
      method: "POST",
      headers: {
        "Content-Type": "application/json",
      },
      body: JSON.stringify({
		"filter":{
			"projectId":4859,
			"startDate":"2024-01-23",
			"endDate":"2024-05-23",
			"dateInterval":"DAY"
			}
		}),
    }
  );
  console.log("res", await res.json());
})();
```

## MOCKDATA + ja-translated Schema
MOCKDATA + ja-translated All wallet analytic endpoints follow a similar request schema
- ja-translated Request: POST
- ja-translated Content-Type: application/json
- MOCKDATA + ja-translated Body (in JSON):
  - MOCKDATA + ja-translated `projectId` (uint64) -- projectID of your project, can be found in the URL of the Builder project.
  - ja-translated `startDate` (timestamp) -- starting date of the query in YYYY--MM--DD format
  - MOCKDATA + ja-translated `endDate` (timestamp) -- ending date of the query in YYYY--MM--DD format
  - MOCKDATA + ja-translated `dateInterval` (OPTIONAL string) -- date interval for the query, options are "DAY", "WEEK", or "MONTH"
- ja-translated Response (in JSON):
  - ja-translated `marketStats` (marketStats\[])
    \[
    - MOCKDATA + ja-translated `value`
      - ja-translated `buyItems` (uint64) -- number of items bought.
      - MOCKDATA + ja-translated `sellItems` (uint64) -- number of items sold
    - MOCKDATA + ja-translated `label` (string) -- label associated with the corresponding endpoint
      ]
