## ja-translated 1. Register a Webhook Listener
MOCKDATA + ja-translated Our filters allow you to listen to on-chain events for particular contract addresses, contract events, specific token ids, account addresses, or topic hashes.

<Callout type="info">
  ja-translated If you require a webhook endpoint to call, you can use [webhook.site](https://webhook.site/) for testing purposes to specify used in the `url` field.
</Callout>

MOCKDATA + ja-translated _Sequence Indexer `AddWebhookListener` Method:_ with `required*` fields
- ja-translated Request: POST /rpc/Indexer/AddWebhookListener
- ja-translated Content-Type: application/json
- MOCKDATA + ja-translated Body (in JSON):
  - MOCKDATA + ja-translated `url*` (string) -- the URL to send the webhook to
  - ja-translated `filters*` (object) -- an object of filters
    - MOCKDATA + ja-translated `contractAddresses*` (\[]string) -- an array of any contract address
    - ja-translated `events*` (\[]string)-- any contract event, with the included `indexed` keyword from the contract when necessary (e.g. `Transfer(address indexed from, address indexed to, uint256 amount)`. Shorthand is also acceptable without argument names, which will simply be parsed as: arg1, arg2, etc. i.e. `Transfer(address indexed,address indexed,uint256)` without argument names)
    - MOCKDATA + ja-translated `tokenIDs` (\[]int) -- an array of token IDs
    - ja-translated `accounts` (\[]string) -- an array of wallet addresses to listen on
    - MOCKDATA + ja-translated `topicHashes` (\[]string) -- a hash of the event being listened to (e.g. ethers.id("Transfer(address indexed from, address indexed to, uint256 amount)"))

<br />

<Callout type="info">
  ja-translated For a full list of events from the various token types: ERC20, ERC721, or ERC1155, [view here](/api/indexer/examples/webhook-listener#on-chain-token-event-types).
</Callout>

	"context"
	"fmt"
	"log"
	"net/http"

	"github.com/0xsequence/go-sequence/indexer"

func GetTransactionHistory(accountAddress string) {

	ok, err := seqIndexer.AddWebhookListener(context.Background(),
		&proto.WebhookListener{
			Url: "https://webhook.site/27c266b7-ee69-4046-8319-75ce91ec2bcf",
			Filters: &proto.WebhookEventFilter{
				Accounts: []prototyp.Hash{prototyp.HashFromString("0xd4Bbf5d234CC95441A8Af0a317D8874eE425e74d")},
                ContractAddresses: []prototyp.Hash{prototyp.HashFromString("0xC852bf35CB7B54a33844B181e6fD163387D85868")},
				// TokenIDs:          []prototyp.BigInt{prototyp.NewBigInt(1)},
			},
		},
	)
	fmt.Println(ok, err)
}

## MOCKDATA + ja-translated 2. Remove a Webhook Listener
ja-translated If you need to clean up your webhook listeners, you can submit requests to remove the listener based on listener `id` and `projectId`:

MOCKDATA + ja-translated _Sequence Indexer `RemoveWebhookListener` Method:_ with `required*` fields
- ja-translated Request: POST /rpc/Indexer/RemoveWebhookListener
- ja-translated Content-Type: application/json
- MOCKDATA + ja-translated Body (in JSON):
  - MOCKDATA + ja-translated `id*` (string) -- the id of the listener returned from `AddWebhookListener` (i.e. `response.listener.id`)
  - ja-translated `projectId*` (string) -- the Project ID the JWT Secret API key was obtained from

<br />

MOCKDATA + ja-translated **Example: `RemoveWebhookListener`**

MOCKDATA + ja-translated :::code-group

```bash [cURL]

  -H "Content-Type: application/json" \
  -H "X-Access-Key: <project_access_key>" \
  -H "Authorization: BEARER <secret_API_key>" \
  -d '{ "id": <listener_id>, "projectId": <project_id> }' \
  https://arbitrum-sepolia-indexer.sequence.app/rpc/Indexer/RemoveWebhookListener
```

	"context"
	"fmt"
	"log"

## ja-translated Webhook Listener Response
MOCKDATA + ja-translated Upon webhook listener response, an object is returned in the following structure
- ja-translated Response (in JSON):
  - ja-translated `uid` (string) -- a deterministic hash value of the on-chain transaction log
  - MOCKDATA + ja-translated `type` (string), -- the type of event (i.e. `BLOCK_ADDED`)
  - ja-translated `blockNumber` (i32) -- the block number from the blockchain for when the event occured
  - MOCKDATA + ja-translated `blockHash` (string) -- the hash of the block for the transaction
  - ja-translated `parentBlockHash` (i32) -- the hash of the parent block
  - MOCKDATA + ja-translated `contractAddress` (string) -- the contract address from where the event came from
  - ja-translated `contractType` (string) -- the type of contract (e.g. `ERC20`, `ERC721`, `ERC1155`, etc.)
  - MOCKDATA + ja-translated `txnHash` (string) -- the transaction hash of the event
  - ja-translated `txnIndex` (i32) -- the index of the transaction in the blockchain block
  - MOCKDATA + ja-translated `txnLogIndex` (string) -- the log index in the transaction
  - ja-translated `ts` (date) -- an ISO 8601 formatted date and time of the event
  - MOCKDATA + ja-translated `event` ([event](/api/indexer/examples/webhook-listener#event-object)) -- the event data of the on-chain data
  - MOCKDATA + ja-translated `txnLogIndex` (string) -- the log index in the transaction
  - ja-translated `logDataType` (string) -- the type of log event (e.g. `TOKEN_TRANSFER`)
  - MOCKDATA + ja-translated `rawLog` ([raw](/api/indexer/examples/webhook-listener#rawlog-object)) -- the raw transaction object

#### ja-translated ERC721

#### MOCKDATA + ja-translated ERC1155

- ja-translated TransferSingle(address indexed operator, address indexed from, address indexed to, uint256 id, uint256 value)
- MOCKDATA + ja-translated TransferBatch(address indexed operator, address indexed from, address indexed to, uint256\[] ids, uint256\[] values)

- MOCKDATA + ja-translated URI(string value, uint256 indexed id)
