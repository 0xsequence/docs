## ja-translated Packaging for Android

### MOCKDATA + ja-translated Android SDK / NDK
ja-translated When setting up your project to build for Android you'll need to update the following settings:
In ProjectSettings/Android SDK
Set SDK API Level to Android-34
Set NDK API Level to anything in the range \[26,33] (We recommend using android-32)

### ja-translated Social Sign In
MOCKDATA + ja-translated **Google:** In order to be able to properly use Google Auth, create and place the Keystore file by following [these instructions](https://dev.epicgames.com/documentation/en-us/unreal-engine/signing-android-projects-for-release-on-the-google-play-store-with-unreal-engine?application_version=5.1).

ja-translated You will also need to generate an \[Android client ID] and a \[Web Application client ID] for your application. And place the \[Web Application client ID] in the `[YourProject/Config/SequenceConfig.ini]`, \[GoogleClientID] field.

MOCKDATA + ja-translated Refer to [these docs](https://developers.google.com/identity/one-tap/android/get-started#api-console) to generate \[Android client ID] and \[Web Application client ID].

ja-translated [This guide](https://https://developers.google.com/android/guides/client-auth?hl=es-419) helps explain how to collect SHA-1 key fingerprints for the \[Android client ID].

MOCKDATA + ja-translated **Apple:** Please ensure you have a proper \[AppleClientID] set in `[YourProject/Config/SequenceConfig.ini]`

## ja-translated Unreal and Xcode Specifics
MOCKDATA + ja-translated During the Unreal Package process in the event a code signing error occurs you can take the following steps within XCode to get your packaged .app file
1. ja-translated After packaging the project in Unreal, open the Xcode project (Sequence-unreal folder -> Intermediate -> ProjectFilesIOS -> SequenceUnreal.xcodeproj)
2. MOCKDATA + ja-translated Click on the project name on the left hand side to open up project settings
3. ja-translated Click the Build Phase Tab
4. MOCKDATA + ja-translated Click on the ‘+’ icon at the top left
5. ja-translated Select Run Script
6. MOCKDATA + ja-translated Drag the new run script to one below from the last item in the phase list
7. ja-translated Expand the run script
8. MOCKDATA + ja-translated In the script box, add the following command: `xattr -cr /[path-to-your-project]/[your-project-name]/Binaries/IOS/Payload/[your-project-name.app]`
9. ja-translated Click on the Build Settings tab
10. MOCKDATA + ja-translated Click on each item under the Architectures header that contains macOS and hit the delete key
11. ja-translated Click on the General tab
12. MOCKDATA + ja-translated Click on Mac and Applevision Pro under supported destinations and hit the delete key
13. ja-translated Now the project can be built (if the build fails at first, wait a few moments then try again. It can sometimes take a bit before the build registers the run script)
14. MOCKDATA + ja-translated Once you have finished running the project, and want to make changes to the code, REMEMBER to delete this xcodeproj file in the sequence-unreal folder to ensure that a new xcodeproj is creating when you packaging the project again.
