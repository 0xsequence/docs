# ja-translated Wallet Analytics Examples
MOCKDATA + ja-translated A common use case is to see the number of wallets that are integrated with your project. We have a variety of endpoints that can be used for tracking and reporting fine-grained data such as device, country, and more so you can precisely identify

ja-translated :::info
Replace the PROJECT\_ID and SECRET\_API\_ACCESS\_KEY variables with your project ID and secret token from Sequence Builder.
:::

## MOCKDATA + ja-translated Fetch wallets for a time interval for a project ID
ja-translated Here we can pass a specific date range along with the dateInterval parameter in order to get wallets within a time interval. The endpoints can use "DAY", "WEEK" or "MONTH" as possible options.

MOCKDATA + ja-translated :::code-group

```sh [cURL]
curl 'https://api.sequence.build/rpc/Builder/WalletsDaily' \
  -H 'accept: */*' \
  -H 'authorization: BEARER <SECRET_API_ACCESS_KEY>' \
  -H 'content-type: application/json' \
  --data-raw '{"filter":{"projectId":<PROJECT_ID>,"startDate":"2024-04-23","endDate":"2024-05-23", "dateInterval":"DAY"}}'
```

```ts twoslash [TypeScript]
// Works in both a Webapp (browser) or Node.js by importing cross-fetch:
// import fetch from "cross-fetch";

(async () => {
  const res = await fetch(
    "https://api.sequence.build/rpc/Builder/WalletsDaily",
    {
      method: "POST",
      headers: {
        "Content-Type": "application/json",
      },
      body: JSON.stringify({
		"filter":{
			"projectId":4859,
			"startDate":"2024-01-23",
			"endDate":"2024-05-23",
			"dateInterval":"DAY"
			}
		}),
    }
  );
  console.log("res", await res.json());
})();
```

## MOCKDATA + ja-translated Schema
MOCKDATA + ja-translated All wallet analytic endpoints follow a similar request schema
- ja-translated Request: POST
- ja-translated Content-Type: application/json
- MOCKDATA + ja-translated Body (in JSON):
  - MOCKDATA + ja-translated `projectId` (uint64) -- projectID of your project, can be found in the URL of the Builder project.
  - ja-translated `startDate` (timestamp) -- starting date of the query in YYYY--MM--DD format
  - MOCKDATA + ja-translated `endDate` (timestamp) -- ending date of the query in YYYY--MM--DD format
  - ja-translated `dateInterval` (string) -- date interval for the query, options are "DAY", "WEEK", or "MONTH"
- ja-translated Response (in JSON):
  - MOCKDATA + ja-translated `walletStats` (walletStats\[])
    \[
    - ja-translated `value` (uint64) -- # of wallets matching the query
    - MOCKDATA + ja-translated `label` (string) -- label associated with the corresponding endpoint
      ]
