# MOCKDATA + ja-translated OIDC - Social Sign In
ja-translated Our SDK supports Social Sign In using [OIDC](https://openid.net/developers/how-connect-works/) with the [implicit flow](https://auth0.com/docs/authenticate/login/oidc-conformant-authentication/oidc-adoption-implicit-flow).

MOCKDATA + ja-translated Currently supported OIDC providers include:
- ja-translated Google -> [Builder Setup](/solutions/builder/embedded-wallet/google-configuration)
- MOCKDATA + ja-translated Apple -> [Builder Setup](/solutions/builder/embedded-wallet/apple-configuration)

ja-translated :::warning
Stop! Have you configured your OIDC providers in the Builder using the instructions linked above?

MOCKDATA + ja-translated Have you added your client id(s) to the `SequenceConfig` scriptable object in your Resources folder?
:::

ja-translated Once the user successfully signs in, an idToken is returned via deep link to the application.
In order to enable deep linking, we must register a custom URL scheme for our application. There are some platform specific requirements to set this up.

### MOCKDATA + ja-translated Editor
MOCKDATA + ja-translated For social sign in to work in the editor, you will need to deploy a simple server on your local machine. We provide a sample implementation [here](https://gist.github.com/BellringerQuinn/d012cd29eb9e756fe3555d4d03b001b2).

ja-translated To deploy this sample server, please perform the following.
1. MOCKDATA + ja-translated Download and install Go: [https://go.dev/dl/](https://go.dev/dl/)
2. ja-translated Open a terminal/command prompt and verify your installation by typing

```
go version
```

3. MOCKDATA + ja-translated Create a directory for your go project and a main.go file

```
mkdir myUnityServer
cd myUnityServer
touch main.go
```

4. ja-translated Copy + paste the [gist](https://gist.github.com/BellringerQuinn/d012cd29eb9e756fe3555d4d03b001b2) into the main.go
5. MOCKDATA + ja-translated Initialize the project by creating a go module

```
go mod init myUnityServer
```

6. ja-translated Run the server

```
go run main.go
```

7. MOCKDATA + ja-translated You should see `Server listening on port 8080` printed to your terminal/command prompt

ja-translated If you choose to deploy the server to somewhere else (a different port, a public server, etc.), you'll want to change the `OpenIdAuthenticator.RedirectUrl` in your code before initiating the sign in process

```csharp
OpenIdAuthenticator.InjectRedirectUrl("the url for your deployed server");
```

MOCKDATA + ja-translated Note: you'll want to use a web client for your social sign in provider with `http://localhost:8080/` (or your server url) whitelisted as a redirect url under `Standalone Platforms` in `SequenceConfig`

### ja-translated iOS

#### ja-translated Sign in with Apple
MOCKDATA + ja-translated This SDK supports native Sign in with Apple via the popular [`apple-signin-unity` package](https://github.com/lupidan/apple-signin-unity). This [requires the `Sign in with Apple` capability in XCode](https://developer.apple.com/documentation/xcode/configuring-sign-in-with-apple#Add-the-Sign-in-with-Apple-capability-to-your-app) in order to function. Provided you have [imported the `Setup` scripts via Package Manager > Samples](https://docs.sequence.xyz/sdk/unity/installation#package-manager---recommended) and/or have `SignInWithApplePostprocessor.cs` under an `Editor` subfolder, this capability will be added to your XCode builds automatically.

ja-translated Please be aware that this capability is not supported by `Personal Teams`.

#### MOCKDATA + ja-translated Other Social Platforms
ja-translated All other social platforms are supported by default, just be sure to add your client IDs in `SequenceConfig`.

### MOCKDATA + ja-translated Android

1. MOCKDATA + ja-translated In the Project window, browse to `Assets > Plugins > Android`.

ja-translated a) Note: in Unity versions 2021.2 and up this path doesn't exist by default. Please navigate to `Edit > Project Settings > Player` and under the Android Publishing Settings, enable `Custom Main Manifest` in the Build section. See [https://docs.unity3d.com/Manual/deep-linking-android.html](https://docs.unity3d.com/Manual/deep-linking-android.html) for more info.
2\. MOCKDATA + ja-translated If it doesn't already exist, create a new file and name it `AndroidManifest.xml`.
3\. ja-translated Paste the following XML into the file, or, if you already have one, add the new keys from this XML to it.
4\. MOCKDATA + ja-translated Make sure to replace 'sdk-powered-by-sequence' with the Url Scheme you set in `SequenceConfig`

```xml
   <?xml version="1.0" encoding="utf-8"?>
   <manifest
     xmlns:android="http://schemas.android.com/apk/res/android"
     xmlns:tools="http://schemas.android.com/tools"
   >
     <application>
       <activity
         android:name="com.unity3d.player.UnityPlayerActivity"
         android:theme="@style/UnityThemeSelector"
       >
         <intent-filter>
           <action android:name="android.intent.action.MAIN" />
           <category android:name="android.intent.category.LAUNCHER" />
         </intent-filter>
         <intent-filter>
           <action android:name="android.intent.action.VIEW" />
           <category android:name="android.intent.category.DEFAULT" />
           <category android:name="android.intent.category.BROWSABLE" />
           <data android:scheme="sdk-powered-by-sequence"/> <!-- substitute 'sdk-powered-by-sequence' with the Url Scheme you set in `SequenceConfig` -->
         </intent-filter>
       </activity>
     </application>
   </manifest>
```

MOCKDATA + ja-translated b) Note: Unity 2022 versions prior to `2022.3.7f1` and 2023 versions prior to `2023.1.7f1`, `2023.2.0b3`, or `2023.3.0a1` contain a bug with Android deep-linking. Please use this XML instead.

```xml
<?xml version="1.0" encoding="utf-8"?>
<manifest
        xmlns:android="http://schemas.android.com/apk/res/android"
        xmlns:tools="http://schemas.android.com/tools"
>
    <application>
        <activity
                android:name="com.unity3d.player.UnityPlayerActivity"
                android:theme="@style/UnityThemeSelector"
                android:exported="true">
            <intent-filter>
                <action android:name="android.intent.action.MAIN" />
                <category android:name="android.intent.category.LAUNCHER" />
            </intent-filter>
            <meta-data android:name="unityplayer.UnityActivity" android:value="true" />
            <intent-filter>
                <action android:name="android.intent.action.VIEW" />
                <category android:name="android.intent.category.DEFAULT" />
                <category android:name="android.intent.category.BROWSABLE" />
                <data android:scheme="sdk-powered-by-sequence"/> <!-- substitute 'sdk-powered-by-sequence' with the Url Scheme you set in `SequenceConfig` -->
            </intent-filter>
        </activity>
    </application>
</manifest>
```

ja-translated i.e. add `<meta-data android:name="unityplayer.UnityActivity" android:value="true" />` after your first `intent-filter`. Please see [https://forum.unity.com/threads/deep-linking-in-unity-2022-the-app-is-restarted.1447300/](https://forum.unity.com/threads/deep-linking-in-unity-2022-the-app-is-restarted.1447300/) for more info.

### MOCKDATA + ja-translated Web
MOCKDATA + ja-translated Social sign in on Web platforms uses React via the [WebGLTemplate](https://docs.unity3d.com/Manual/webgl-templates.html) found under `WebGLTemplates/SequenceReact`. This has been validated on WebGL and WebGPU. To learn how to enable WebGPU in Unity, please see [this forum post](https://forum.unity.com/threads/early-access-to-the-new-webgpu-backend-in-unity-2023-3.1516621/).

ja-translated Please note that Google is the only OIDC provider currently supported on Web platforms by our default `SequenceReact` WebGLTemplate; however, you are welcome to expand upon it to add support for the other OIDC providers supported by our API.

#### MOCKDATA + ja-translated One-time setup
ja-translated Drag this `WebGLTemplates` folder under `Assets` (or just drag the `SequenceReact` template into your existing `WebGLTemplates` folder is you already have one) such that you have `Assets/WebGLTemplates/SequenceReact`.

MOCKDATA + ja-translated :::warning
It is very important that your `WebGLTemplates` are at `Assets/WebGLTemplates`. Much like the `Plugins` folder, if they aren't in this exact location, Unity will not pick them up!
:::

ja-translated Once you've added the `SequenceReact` template, you'll want to restart the editor so that Unity picks up your newly added template.

MOCKDATA + ja-translated From here, you can select the template under `Project Settings > Player Settings > Resolution and Presentation`.

#### ja-translated Building
MOCKDATA + ja-translated After the one time setup, when you build your Unity project on Web platforms it will be built as part of a React project.
1. ja-translated Open up a command line at the root of your new build
2. MOCKDATA + ja-translated [Install pnpm](https://pnpm.io/installation) if you don't already have it
3. ja-translated `pnpm install` to install dependancies
4. MOCKDATA + ja-translated `pnpm dev` to run the project on your localhost (defaulting to port 4444 - configurable in `SequenceReact/package.json`)
5. ja-translated Open [http://localhost:4444](http://localhost:4444)

MOCKDATA + ja-translated :::tip
When configuring your social sign in credentials with Google (for example), you'll want to whitelist [http://localhost:4444](http://localhost:4444) and your game's eventual hosted URL as "Authorized Javascript origins"
:::

MOCKDATA + ja-translated :::tip
Don't forget to subscribe to the `SequenceWallet.OnWalletCreated` event to receive your newly created wallet!
:::
