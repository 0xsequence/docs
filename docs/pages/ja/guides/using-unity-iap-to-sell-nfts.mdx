# ja-translated Using Unity IAP to Sell NFTs
MOCKDATA + ja-translated Most of your players will not have cryptocurrencies in their wallets. Onboarding funds into cryptocurrencies using traditional means can be a hassle for players and may be too stressful for users who have less technical experience. In any case, minimizing payment friction has always been a very important requirement to an effective monetization strategy. The most frictionless method for making microtransactions for the majority of users on mobile will be with the in-app purchasing method they are already used to: IAP via the iOS App Store and Android Google Play Store. This is especially true if your are using Embedded Wallets and building games where the blockchain is invisible.

ja-translated This guide will explain how we updated [Jelly Forest, our embedded wallet showcase game made with Unity](/guides/jelly-forest-unity-guide), to add an IAP that mints you a goofy hot dog hat for your Jelly.

## MOCKDATA + ja-translated Deploy a Smart Contract
ja-translated The first step, if you haven't done so already, is to [deploy a smart contract](/solutions/collectibles/contracts/deploy-an-item-collection) that will define and represent the NFTs you aim to sell to your players and display in your game.

MOCKDATA + ja-translated Once you've deployed your smart contract, don't forget to [add your contract address as a Sponsored Address on the "Gas Sponsoring"](/solutions/builder/gas-tank#gas-tank-in-builder) page on the Builder Console! This will make it so that your users have their gas fees automatically sponsored using your compute credits when interacting with your game's smart contracts.

## MOCKDATA + ja-translated Deploy a Remote Minter
ja-translated By default, ERC1155/721 contracts deployed via the Builder Console require callers to have the appropriate permissions in order to mint a token. While this may seem like a nuisance at first glance, this is a good thing! Without this, anyone could call the mint method on your contract and give themselves infinite in-game items!

ja-translated You'll want to deploy a server with a Sequence wallet (or other) and give it minting permissions in the builder.

ja-translated In Jelly Forest, all the coins you collect during gameplay are minted as ERC1155 tokens. Here's how we did it:

MOCKDATA + ja-translated and login

7. ja-translated Open `wrangler.toml`
   1. MOCKDATA + ja-translated Give your server a name by changing the `name` string

Metamask can be used to easily [setup a wallet](https://support.metamask.io/hc/en-us/articles/360015489531-Getting-started-with-MetaMask) and [export the private key](https://support.metamask.io/hc/en-us/articles/360015289632-How-to-export-an-account-s-private-key). 11. MOCKDATA + ja-translated In the command line where the localhost server is running, you should see that the minter's wallet address has been logged

### MOCKDATA + ja-translated Handle IAP Purchased Tokens with the Remote Minter
ja-translated Above, we've deployed a remote minter that is useful for minting tokens that can be earned through in-game actions. Let's see how we can leverage this server so that we can also mint tokens that are purchasable via IAP.

MOCKDATA + ja-translated You'll notice that a receipt can also be included in the payload sent to the remote minter. This is where you'll include the IAP receive from Google/Apple. [Unity recommends](https://docs.unity3d.com/Packages/com.unity.purchasing@4.0/manual/BackendReceiptValidation.html) using [Nobuyori Takahashi's IAP project](https://github.com/voltrue2/in-app-purchase) to verify IAP receipts received via Unity IAP on the server side.

ja-translated Once you've validated the receipt on the server side, you can proceed with your minting logic, using the provided sample code above as a reference.

## MOCKDATA + ja-translated Unity Implementation
ja-translated On the Unity side, the first step is integrating [Unity IAP](https://docs.unity3d.com/Manual/UnityIAP.html) into your project.

MOCKDATA + ja-translated In your `ProcessPurchase` methods in your `IStoreListener` from the integration process, you'll want to initiate the minting process. In Jelly Forest, this is done via the [UnityIAP](https://github.com/0xsequence/sequence-unity-demo/blob/JellyForest/Scripts/UnityIAP.cs) and [PremiumItem](https://github.com/0xsequence/sequence-unity-demo/blob/JellyForest/Scripts/Items/PremiumItem.cs) scripts.

ja-translated In terms of the payload, you can see in the `PremiumItem` implementation that I am adding a PermissionedMintTransaction to a `TransactionQueuer`.

```csharp
public void AddToPremiumTransactionQueue(PermissionedMintTransaction payload, string iapReceipt)
{
    PremiumIAPMinter minter = new PremiumIAPMinter(new MintingRequestProver(Wallet, Chain),
        _mintEndpoint, ContractAddress, iapReceipt);
    _permissionedMinterTransactionQueuer.Enqueue((payload, minter));
}
```

MOCKDATA + ja-translated where `_permissionedMinterTransactionQueuer` is a `PermissionedMinterTransactionQueuer`.

ja-translated This will send a payload in this format:

```json
ProofPayload: 
{
    "app": "Made with Sequence Unity SDK App",
    "iat": (uint)DateTimeOffset.UtcNow.ToUnixTimeSeconds(), // issued at time 
    "exp": (uint)DateTimeOffset.UtcNow.ToUnixTimeSeconds() + 300, // expiry time 
    "ogn": "Sequence Unity SDK",
    "payload": {
        "contractAddress": "0xabc123...",
        "tokenId": "11",
        "amount": 5,
        "receipt": <IAP Receipt String here>
    }
}

This JSON get stringified and included in the MintingRequestProof:
{
    "Proof": "{\"app\": \"Made with Sequence Unity SDK App\", \"iat\": ...}",
    "SignedProof": "0x123def...", // proof signed by the player's embedded wallet
    "SigningAddress": "0xa1b2c3..." // the player's embedded wallet address
}
```

MOCKDATA + ja-translated For more information on the transaction queuers, please checkout [this doc](/sdk/unity/write-to-blockchain#transaction-queuers).

ja-translated :::warning
Apple and Google allow users the option to chargeback an IAP made by mistake. In this case, your user will still have the token(s) minted to them. While you can handle this via traditional means (e.g. banning players who frequently chargeback purchases), you should be careful not to sell high-value NFTs via this method due to fraudulent chargeback risk.
:::
