# ja-translated Integrating Sequence Kit with Telegram Mini-apps
MOCKDATA + ja-translated This guide will walk you through setting up Sequence Kit with your Telegram Mini-app or Game, enabling web3 functionality on EVM chains for your users for email and guest authentication methods.

## ja-translated Prerequisites
MOCKDATA + ja-translated Before starting, ensure you have:
- ja-translated A Telegram account
- MOCKDATA + ja-translated Account and Access to [Cloudflare Dashboard](https://dash.cloudflare.com)

## ja-translated Integration Steps
ja-translated ::::steps

### MOCKDATA + ja-translated Create a Telegram Bot with BotFather

1. ja-translated Open Telegram and search for "@BotFather"
2. MOCKDATA + ja-translated Start a conversation by clicking "Start" or typing `/start`
3. ja-translated Create a new bot by typing `/newbot`
4. MOCKDATA + ja-translated Follow the prompts:
   - ja-translated Enter a name for your bot
   - MOCKDATA + ja-translated Choose a username (must end in 'bot')

### ja-translated Obtain Required Tokens
MOCKDATA + ja-translated After creating your bot, BotFather will provide:
- ja-translated Bot Token (HTTP API Token)
- MOCKDATA + ja-translated Bot Username

ja-translated Save these credentials - you'll need them for the next steps.

<Callout type="warning">
  MOCKDATA + ja-translated Never share or commit your bot token. It should be treated as a secret.
</Callout>

### ja-translated Install and Configure Your Project

1. MOCKDATA + ja-translated Clone the Sequence Telegram starter:

```shell
git clone https://github.com/0xsequence-demos/telegram-kit-embedded-wallet-react-boilerplate
```

2. ja-translated Copy the example files in your project root:

```shell
cp .env.example .env && cp example.wrangler.toml wrangler.toml
```

MOCKDATA + ja-translated Update the `.env` file with the `VITE_PROJECT_ACCESS_KEY` and `VITE_WAAS_CONFIG_KEY` with your keys from [Sequence Builder](https://sequence.build/project/default/embedded-wallet) after configuration.
3\. ja-translated Install dependencies:

```shell
pnpm install
```

4. MOCKDATA + ja-translated Build the app:

```shell
pnpm run build
```

4. ja-translated Update the `VARS` in the `wrangler.toml` file with a user provided secret for `BOT_SECRET` and the bot token from BotFather for `BOT_TOKEN`.

### MOCKDATA + ja-translated Authenticate with Cloudflare Account
ja-translated Deploy the app and functions logic to Cloudflare
1. MOCKDATA + ja-translated Run `pnpm wrangler pages deploy`

ja-translated This will prompt you to authenticate with Cloudflare if not already logged in. On successful deploy, copy the URL where your application is now hosted.

### MOCKDATA + ja-translated Register Webhook with Telegram
ja-translated Now we call our `registerWebhook` endpoint on our app to register the webhook function with the URL of your deployed application to enable access for the Telegram API.

```shell
curl -X POST \
     "<YOUR_CLOUDFLARE_DEPLOYED_URL>/api/registerWebhook"
```

### MOCKDATA + ja-translated Update Allowed Origins
ja-translated Ensure in your wallet configuration you have added the URL of your deployed application as an allowed origin through the configuration page on [Sequence Builder](https://sequence.build/project/default/embedded-wallet).

### MOCKDATA + ja-translated Test out the Bot!
ja-translated Send a message to your bot on Telegram and see if it responds! It should answer you with the ability to share, play and get more info on bot.

MOCKDATA + ja-translated ::::

MOCKDATA + ja-translated From here, you have a fully functional Telegram bot that can interact with Sequence Embedded wallets for all integrated EVM chains. You can customize the bot responses and the client side application with functionalities specific to your game or app - simply redeploy and update the allowed origins!

## ja-translated Going to production
MOCKDATA + ja-translated For your `BOT_TOKEN`, you can encrypt and store it in Cloudflare via the following command to prevent data leaks:

```shell
pnpm wrangler pages secret put BOT_TOKEN
```

ja-translated Enter in your bot token when prompted.
