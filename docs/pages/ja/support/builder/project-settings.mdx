# ja-translated Settings in Sequence Builder

## MOCKDATA + ja-translated Settings Options
ja-translated There are various actions available for your project in Settings of Sequence Builder.

MOCKDATA + ja-translated The Settings section in Sequence Builder is categorized into six sections:
1. ja-translated **General**: Update your [project name](/support/builder/project-settings#project-name) or [avatar](/support/builder/project-settings#avatar)
2. MOCKDATA + ja-translated **Advanced**: Update your [theme](/support/builder/project-settings#2-advanced-settings)
3. ja-translated **Collaborators**: Add [collaborators](/support/builder/project-settings#add-a-collaborator) or create [invite links](/support/builder/project-settings#create-an-invite-link)
4. MOCKDATA + ja-translated **Networks**: Update the [networks](/support/builder/project-settings#3-network-settings) your project is using
5. ja-translated **API Access Keys**: Copy [keys](/support/builder/project-settings#copy-your-access-key) and [configure existing keys](/support/builder/project-settings#update-api-access-settings-of-a-key)
6. MOCKDATA + ja-translated **Billing**: Update your [project plan](/support/builder/project-settings#update-your-project-subscription-plan) or set [overage limits](/support/builder/project-settings#set-your-overage-limit)

ja-translated The final action you can take in settings is to [delete your project](/support/builder/project-settings#delete-a-project).

MOCKDATA + ja-translated Settings are configured per project, so let's start by navigating to the settings of a specific project.

## ja-translated 2. Collaborator Settings
MOCKDATA + ja-translated Collaborators for any project can be added and deleted.

ja-translated You can either follow this below section, or, skip ahead to the [video](/support/builder/project-settings#watch-how-to-invite-collaborators) for inviting collaborators.

MOCKDATA + ja-translated Start, by opening your Collaborators settings.

ja-translated ![Builder settings select Collaborators](/img/builder/builder_settings_collaborators.png)

MOCKDATA + ja-translated In the Collaborator settings, you have two methods to add collaborators to your project.

ja-translated The first method is to directly add collaborators by entering their information and selecting their access level.

MOCKDATA + ja-translated The second method is to create an invite link. When using this option, you can specify the access level, link duration, and optional usage limits.

ja-translated Let's walk through each of these.

### MOCKDATA + ja-translated Add a Collaborator
MOCKDATA + ja-translated To add a collaborator, make sure you have selected the [correct project](/support/builder/project-settings#select-a-project), accessed [Settings](/support/builder/project-settings#go-to-settings), and are in the [Collaborators](/support/builder/project-settings#2-collaborator-settings) section.

ja-translated Start by clicking on the `Add Collaborator` button.

MOCKDATA + ja-translated Enter a wallet address in the user address field, and then select the access level you want using the role dropdown.

ja-translated Finish by clicking the `Add new Collaborator` button.

MOCKDATA + ja-translated ![Builder settings Collaborators Add Collaborator modal](/img/builder/builder_settings_add_collaborator_modal.png)

ja-translated To confirm, you should see your new collaborator in the dashboard, including their wallet address and access level.

MOCKDATA + ja-translated ![Builder settings Collaborators confirm collaborator](/img/builder/builder_post_add_collaboration.png)

### ja-translated Create an Invite Link
MOCKDATA + ja-translated Start by clicking on the `Create Invite Link` button and the following modal should appear

ja-translated ![create invite link](/img/builder/builder_settings_create_invite_link.png)

MOCKDATA + ja-translated Then, select the access level you want your invite link to give: `Read`, `Write`, or `Admin`.

ja-translated Enter the number of collaborators you want this link to be limitted to (setting a limit is optional).

MOCKDATA + ja-translated Finally, select the amount of time you want this link to be valid: a day, 3 days, or a week.

ja-translated Finish by clicking the `Create Link` button.

MOCKDATA + ja-translated To confirm, you should see your invite link in the dashboard, including the access level, limit, date created, and time remaining until it expires.

ja-translated ![Builder settings Collaborators confirm invite link](/img/builder/builder_invite_link_confirmation.png)

#### MOCKDATA + ja-translated Sharing an Invite Link
ja-translated To share an invite link, click the `Copy Link` button.

MOCKDATA + ja-translated Go to where you want to share the link, paste it, and send it to your collaborators.

#### ja-translated Deleting an Invite Link
MOCKDATA + ja-translated To remove an invite link, click the `Delete Link` button.

ja-translated You will be prompted to confirm the deletion; click `Yes` to delete.

MOCKDATA + ja-translated After deleting the link, you'll get a success notification at the bottom right of your dashboard, and the link will no longer be visible.

## ja-translated 4. API Access Keys Settings
MOCKDATA + ja-translated Open your API Access Keys settings.

ja-translated ![Builder settings select API Access Keys](/img/builder/builder_settings_select_access_keys.png)

MOCKDATA + ja-translated In the API Access Keys section, you will find a dashboard with several options: monitor the usage of each key, set your default key, or expand the details of a specific key.

ja-translated You can also add a new key or edit the details of an existing key.

MOCKDATA + ja-translated For each key, there are specific actions you might want to take, such as:
- ja-translated [Rename](/support/builder/project-settings#renaming-your-key) or [delete](/support/builder/project-settings#deleting-your-key) your key
- MOCKDATA + ja-translated [Copy](/support/builder/project-settings#copy-your-key) or [change](/support/builder/project-settings#rotate-your-key) the code of your key
- ja-translated Configure [how](/support/builder/project-settings#set-service-access) and [where](/support/builder/project-settings#set-domain-access) your key can be used

### MOCKDATA + ja-translated Rename or Delete an API Access Key
ja-translated To rename or delete one of your keys, make sure you have selected the [correct project](/support/builder/project-settings#select-a-project), accessed [Settings](/support/builder/project-settings#go-to-settings), and are in the [API Access Key](/support/builder/project-settings#4-api-access-keys-settings) section.

MOCKDATA + ja-translated Start by selecting the key you want to edit.

ja-translated ![Builder settings API Access Keys select API key for rename](/img/builder/builder_settings_set_dev_key.png)

##### MOCKDATA + ja-translated Renaming your Key
ja-translated In the name field, and enter a new name.

MOCKDATA + ja-translated Then click the `Update` button.

ja-translated ![Builder settings API Access Keys rename key](/img/builder/builder_rename_key.png)

##### MOCKDATA + ja-translated Deleting your Key
MOCKDATA + ja-translated Click the `Delete` button in the bottom left.

ja-translated ![Builder settings API Access Keys delete key](/img/builder/builder_settings_delete_a_key.png)

ja-translated You will be prompted to confirm the deletion; click `Yes` to delete.

##### MOCKDATA + ja-translated Copy your Access Key
ja-translated Your API key is designed to be used in another location or application. To use it, you need to copy it and paste it into your application or code.

MOCKDATA + ja-translated ![copy Builder settings API Access Key](/img/builder/builder_settings_copy_key.png)

ja-translated To copy your access key, click `Copy` in the Access Key field.

MOCKDATA + ja-translated After clicking, you'll get confirmation when the button briefly changes to `✓ Copied`.

ja-translated Your project access key is now copied to your clipboard.

##### MOCKDATA + ja-translated Rotate your Access Key
ja-translated You may want to change your access key, if for some reason your key was leaked, or, your preference is to observe what generates as context for your project

MOCKDATA + ja-translated By rotating your key you will not lose any of your project's usage history

ja-translated To do this, click the `Rotate` button in the Access Key field

MOCKDATA + ja-translated After clicking, you'll see the key transform with a success notification in the bottom right.

ja-translated ![Builder settings API Access Keys rotate key](/img/builder/builder_settings_rotated_confirmation.png)

### MOCKDATA + ja-translated Update API access settings of a Key
ja-translated To update API access settings of a key, make sure you have selected the [correct project](/support/builder/project-settings#select-a-project), accessed [Settings](/support/builder/project-settings#go-to-settings), and are in the [API Access Key](/support/builder/project-settings#4-api-access-keys-settings) section.

MOCKDATA + ja-translated Start by selecting the key that you want to update the access to.

ja-translated ![Builder settings API Access Keys select API key for rename](/img/builder/builder_settings_set_dev_key.png)

##### ja-translated Set Domain Access
MOCKDATA + ja-translated You can ensure that the services linked to your API key are only utilized on domains connected to your game or app by enabling specific domains where your keys can be used.

ja-translated To do this, enter a domain you want to enable access to into the domain field.

MOCKDATA + ja-translated If you want to include another domain, click the `Add Domain` button and add it too. You should now have a list of domains your key can be used with.

ja-translated Continue adding to this list until you have included all of the domains that you need.

MOCKDATA + ja-translated Complete your update by clicking the `Update button` to save.

ja-translated ![Builder settings API Access Keys add domains](/img/builder/builer_settings_set_domain.png)

##### MOCKDATA + ja-translated Set Service Access
ja-translated You can set your API key to enable only specific services rather than all the services we offer.

MOCKDATA + ja-translated To do this, look near the bottom right you will find several services listed with checkboxes.

ja-translated Check the box for each service that you want your key to have access to.

MOCKDATA + ja-translated Complete your update by clicking the `Update button` to save.

MOCKDATA + ja-translated ![Builder settings API Access Keys check services](/img/builder/builder_settings_update_services.png)

## MOCKDATA + ja-translated Delete a Project

The following instructions are destructive and irreversible, so do not follow these steps unless you intend to delete your project.

MOCKDATA + ja-translated To delete your project, make sure you have selected the [correct project](/support/builder/project-settings#select-a-project) and are in the [Settings](/support/builder/project-settings#go-to-settings) section. You will also need administrative-level permissions to delete a project.

ja-translated If you are an Admin, you will notice a section labeled <span style={{color: "#c2501f"}}>Danger Zone</span> under all your settings sections.

MOCKDATA + ja-translated In this area, you will also find an option to `Delete this Project` — click that option.

ja-translated ![Builder delete project](/img/builder/settings_builder_delete_project.png)

MOCKDATA + ja-translated This action will prompt a modal to confirm that you want to delete your Project.

ja-translated ![Builder delete project](/img/builder/builder_settings_input_delete_project_name.png)
