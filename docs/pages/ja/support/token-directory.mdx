# ja-translated Token Directory
MOCKDATA + ja-translated When tokens are added to various blockchains, the default is for these tokens to not be included in the default wallet settings since a lot of tokens can be considering spam until they are verified.

ja-translated The best way to get your contracts verified is to deploy them through the Sequence Builder which will add them to our token directory automatically.

MOCKDATA + ja-translated We also support verifying external contracts and collections, to do this - please get in [contact](https://go.crisp.chat/chat/embed/?website_id=9ef4395b-6bb1-4645-8e02-6071d89290a1).

ja-translated Otherwise, if you are deploying tokens for testing - you can also configure your Sequence Wallet to show unverified tokens.

## MOCKDATA + ja-translated How to make tokens appear in your wallet
ja-translated From your Sequence Wallet in Builder, navigate to [Settings > General](https://sequence.app/settings/general) and untoggle `Hide Unverified Tokens`.

<Callout type="warning">
  MOCKDATA + ja-translated Note <br />
  This will show all unverified tokens which can be utilized for testing purposes.
</Callout>
