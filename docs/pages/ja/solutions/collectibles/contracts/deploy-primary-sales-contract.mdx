### ja-translated How to Deploy a Primary Sales Contract in Sequence Builder
MOCKDATA + ja-translated This guide walks through how to setup and deploy the contracts for launching a primary sale - suitable for an NFT Drop, in-game store, and more.

ja-translated ::::steps

### ja-translated Step 1: Navigate to contracts
MOCKDATA + ja-translated Start by selecting your desired `project` you would like to create a sale for and head to `Contracts > Deploy`.

### ja-translated Step 2: Deploy your chosen collectible contract
MOCKDATA + ja-translated Follow this [guide](/solutions/collectibles/contracts/deploy-an-item-collection) in order to deploy a collectible contract as well as upload the collection metadata that you wish to create a sale for.

MOCKDATA + ja-translated ![select game item](/img/collectibles/select_web3_game_item.png)

### ja-translated Step 3: Deploy your Primary Sales contract
MOCKDATA + ja-translated Once successfully deployed, return to the contracts page and select `+ Deploy new contract` again. Then select the appropriate ERC721 or ERC1155 sale contract that corresponds to your previously deployed collectible contract in step 2 and click `Deploy sale contract`.

ja-translated ![Deploy Sale Contract](/img/collectibles/primary-sales/docs_deploy_sale.png)

### MOCKDATA + ja-translated Step 4: Set your mint details on the Primary Sales contract.
ja-translated Navigate to the mint details tab for your primary sales contract. Fill out the mint details with your desired sale parameters.

MOCKDATA + ja-translated ![Mint Access](/img/collectibles/primary-sales/docs_mint_access.png)
- ja-translated `Supply Cap`: This is the amount of supply of tokens that users can mint via the Sale contract
- MOCKDATA + ja-translated `Cost`: Is the cost amount in GWEI of minting a single token. In this example, we set it to 0 to easily mint from Builder, but when deploying in production this should match what you wish to charge your users.
- ja-translated `Start and End Time`: Start and end time of the sale. If the current time is within the date range provided, this will activate the sale. Consider setting the date range to the future if desired.
- MOCKDATA + ja-translated `Payment Token`: You can select the native token on the chain or a custom currency as well that the sale is in.
- ja-translated `Sale Type`: You can also opt to choose between a Public or Private mint.  When selecting a private mint, you can also choose a list of allowlisted participants that is uploaded using our [Audience](https://sequence.build/project/default/audience/) feature. Important to note, that for a private mint you will need to pass in a merkle proof that corresponds to the merkle root added in the sale details in order to mint. Not to worry though, we provide this logic in our boilerplates by default.

MOCKDATA + ja-translated For this walkthrough, we will assume you are creating a public mint.

### ja-translated Step 5: Add MinterRole for Sales Contract to Collection Contract
MOCKDATA + ja-translated Copy the Sales contract address, then you can navigate to the specific contract and select the settings to view Permissions.

ja-translated ![contract settings](/img/builder/builder_contract_specific_settings.png)

MOCKDATA + ja-translated Once you have the modal open, select the `Permissions` tab and you can `Edit`, or, `+ Add Collaborator`.

ja-translated ![add collaborator](/img/builder/builder_contract_specific_permissions.png)

MOCKDATA + ja-translated Then complete the form with the Sales contract address to add as a collaborator and select the dropdown to assign the `Minter` role.

ja-translated ![assign role](/img/builder/builder_contract_permissions_dropdown.png)

MOCKDATA + ja-translated Finally, complete and sign the transaction update the contract Access Control.

### ja-translated Step 6:
MOCKDATA + ja-translated Now that you've got your contracts configured, let's try to do a test mint!

ja-translated Return to your deployed Sale contract and select `Write contract`. We will then click on the `mint()` function and pass in a few parameters:
- MOCKDATA + ja-translated `to`: this is the address that the NFT is being minted to. Generally this would be the user who paid, but we can simply pass in our Builder address located in the top-right hand corner.
- ja-translated `amount`: The amount of NFTs to mint, we will simply mint 1.
- MOCKDATA + ja-translated `paymentToken`: For a custom currency, you would pass in that currency's address. However for native tokens it will be `0x0000000000000000000000000000000000000000`.
- ja-translated `maxTotal`: The maximum amount of the given currency that can be paid.
- MOCKDATA + ja-translated `proof`: For a public mint, you can simply leave this field blank. In a private mint, this would be the generated proof that corresponds to the user's address & previously provided merkle root.

ja-translated ![Grant Minter Role](/img/collectibles/primary-sales/docs_mint_sale.png)

MOCKDATA + ja-translated Once all that information is filled out, go ahead and click `Write` to build the transaction, then `Confirm` to mint the token!
