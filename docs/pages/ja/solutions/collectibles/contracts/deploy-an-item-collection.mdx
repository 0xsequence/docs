# ja-translated How to Deploy an Item Collection Contract in Sequence Builder
MOCKDATA + ja-translated This guide walks through how to setup and deploy a Web3 Game Item contract in Builder.

<Callout type="warning">
  ja-translated Prerequisite: Create a Project

  This guide assumes that you have already [signed up for Builder and created a Project](/solutions/builder/getting-started).
</Callout>

#### MOCKDATA + ja-translated ERC721 vs. ERC1155
ja-translated Both contracts mint NFTs, but ERC721, being the earlier standard, has gained widespread adoption, particularly in digital collectibles. Known for its simplicity and ease to audit for security, ERC721 tokens are distinct, contributing to their recognition in various NFT marketplaces. Typically you would use an ERC721 contract if: you're dealing with the single same asset, multiplied, or a multiplicity of 1 item with many in a collection

MOCKDATA + ja-translated Example `ERC721`: [Azuki](https://etherscan.io/token/0xed5af388653567af2f388e6224dc7c4b3241c544)

ja-translated In contrast, ERC1155 offers versatility with efficient batch operations for both fungible and non-fungible tokens in a single contract, where typically you would be dealing with a multiplicity of a many items to many quanties relationship. However, this flexibility introduces complexity, necessitating careful security attention.

MOCKDATA + ja-translated Example `ERC1155`: [Skyweaver](https://polygonscan.com/token/0x631998e91476da5b870d741192fc5cbc55f5a52e)

## ja-translated Step 1: Navigate to contracts
MOCKDATA + ja-translated Start by selecting your `project` in the top left and corner for what you want to create the collectible for, and head to the `contracts` section and select `+ Deploy new contract`

ja-translated ![select project and new contract](/img/collectibles/deploy_new_contract.png)

## MOCKDATA + ja-translated Step 2: Choose your collectible type
ja-translated Then make your choice of Web3 Game Item Collection (ERC1155) or NFT Collection (ERC721), for this example we'll walk you through a Web3 Game Item Collection (ERC1155)

MOCKDATA + ja-translated ![select game item](/img/collectibles/select_web3_game_item.png)

<Callout type="warning">
  ja-translated The only difference in deployment between an `ERC1155` vs `ERC721` is that you
  add a `symbol` to an ERC721 NFT Collection
</Callout>

## MOCKDATA + ja-translated Step 3: Specify contract details
ja-translated Complete the contract details by specifying a `Contract Name` and `Owner` for your contract, with the option to input Royalties. Make sure the Owner address is the Sequence Wallet in the top-right hand corner and that you have funds in this wallet on mainnet, otherwise on testnet - we sponsor these transactions for you.

MOCKDATA + ja-translated ![deploy game item](/img/collectibles/deploy_game_item.png)

<Callout type="danger">
  ja-translated Note:

  While you can change the `name` later in the builder interface where it will update across the sequence stack, popular explorers do not reindex the information, so what you put first remains
</Callout>

## MOCKDATA + ja-translated Step 4: Deploy your contract
ja-translated Deploy your contract from the popup window at [http://sequence.app/sign-transaction](http://sequence.app/sign-transaction), and sign the message by selecting `confirm`

<img src="/img/collectibles/sign_deploy_transaction.png" width="200" alt="Deploy your contract by signing the message in the popup window from the Sequence Wallet" />

<Callout type="info">
  MOCKDATA + ja-translated All testnets will be free to transact on.
</Callout>

### ja-translated Optional: Add a Minter Role to the contract
MOCKDATA + ja-translated Sequence contracts have Access Control options by default. In some cases, you may want to grant a specific address permissions in order to call various functions such as `mint()`.

ja-translated First start by selecting your contract you just deployed in the `contracts` section

MOCKDATA + ja-translated ![select contract](/img/collectibles/select_deployed_contract.png)

ja-translated Next, head to the `Write Contract` section

MOCKDATA + ja-translated ![write contract](/img/collectibles/select_item_write_contract.png)

ja-translated In the `grantRole` section of the write contract tab navigation

MOCKDATA + ja-translated ![grant role](/img/collectibles/grant_role_game_item.png)

ja-translated Complete with the following details:

MOCKDATA + ja-translated `bytes32 role`: `0x9f2df0fed2c77648de5860a4cc508cd0818c85b8b8a1ab4ceeef8d981c8956a6`
`address account`: `<Wallet Address>`

ja-translated Where the wallet address is the address that you would like to give permissions to mint to, specifically any Sequence wallet.

MOCKDATA + ja-translated ![copy address](/img/collectibles/copy_address_game_item.png)

ja-translated Complete by selecting `write` and signing the transaction in the popup window, like before

MOCKDATA + ja-translated With a confirmation message showing `Response Success`

<Callout type="info">
  ja-translated The role string inputted is the result of `solidityPackedKeccak256("MINTER_ROLE")` in
  solidity or `ethers.solidityPackedKeccak256(ethers.toUtf8Bytes("MINTER_ROLE"))`
  in javascript
</Callout>

<Callout type="tip">
  MOCKDATA + ja-translated It's a typical pattern to use offchain compute like a cloudflare worker with a
  relayer wallet sending transactions, which would be inputted in the account
  field, more on how to accomplish this
  [here](/guides/mint-collectibles-serverless#mint-collectibles-using-a-gasless-serverless-transactions-api)
</Callout>

## ja-translated Step 6: Mint tokens to your wallet address
MOCKDATA + ja-translated Navigate to the `mint` card in the `Write Contract` section and input the `to` being the wallet address you would like to receive the token to, the `tokenId` (typically starting at 0), and `amount` of tokens, and finally the `data` section you can just input `0x00`, which typically represent Additional data with no specified format.

ja-translated ![mint tokens](/img/collectibles/mint_game_item.png)

## MOCKDATA + ja-translated Step 7: Confirm your minted collectible
ja-translated And you're done!

MOCKDATA + ja-translated You can view the transactions submitted to the blockchain for your wallet address in the `Transactions` tab navigation

ja-translated ![view currency transactions](/img/collectibles/transactions_game_items.png)
