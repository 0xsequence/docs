# MOCKDATA + ja-translated Get Started

ja-translated Signing up and creating a project is easy to do with Sequence Builder and should take you less than a few minutes. In this tutorial, we will help you sign up for Builder and then create your first project.

MOCKDATA + ja-translated Let's get started!

## ja-translated Sign up for Sequence Builder

ja-translated ::::steps

### MOCKDATA + ja-translated Go to sequence.build

ja-translated Go to [Sequence.Build](https://sequence.build/) and click on the `Start Building` or `Sign in` button to get started.

MOCKDATA + ja-translated ![Sequence builder landing page](/img/builder/builder_landing_page_signup_project.png)

### ja-translated Choose a sign in method

MOCKDATA + ja-translated When you first sign in to Builder, a wallet is created for you.

ja-translated You may, however, already have a Sequence wallet. If you are signed in with that wallet, you will see an option to use it.

MOCKDATA + ja-translated Without a wallet, you will be given a few options to sign in. You can use any of the social sign ins, or you can simply use your email to get started.

MOCKDATA + ja-translated ![Sequence builder choose signin](/img/builder/builder_choose_signin_signup_project.png)

### ja-translated Sign in to sequence.build

MOCKDATA + ja-translated After choosing how you would like to sign in, you will be presented with the terms of use. To read the `Terms of Use`, `Privacy Policy`, and `Master Services Agreement`, click on the respective links for each.

ja-translated ![Sequence builder accept terms](/img/builder/builder_accept_terms_signup_project.png)

MOCKDATA + ja-translated Once you have read each, click through to continue.

ja-translated You are now signed in to Builder!

## MOCKDATA + ja-translated Create a Project

MOCKDATA + ja-translated Everything you do in Builder is centered around each project - so let's start one.

ja-translated The easiest way to start a project is to simply click `+ Create Project`.

MOCKDATA + ja-translated ![Sequence builder dashboard](/img/builder/builder_dashboard_signup_project.png)

### ja-translated Enter the project details

MOCKDATA + ja-translated A modal will pop up, allowing you to enter details about your project. Provide a project name and, if you like, add an avatar for the project.

ja-translated Then, select the network(s) on which you would like to start deploying. This can always be changed later.

MOCKDATA + ja-translated ![Sequence builder enter details](/img/builder/builder_enter_details_signup_project.png)

### ja-translated Create the project!

MOCKDATA + ja-translated Now hit the `Create new Project` button!

ja-translated ![Sequence builder project created](/img/builder/builder_project_created_signup_project.png)

MOCKDATA + ja-translated Once you finish, you should now be viewing your project dashboard.

### ja-translated Claim an API Access Key

MOCKDATA + ja-translated Finally, to start building you'll need to claim an access key from the Builder, which can be accomplished by navigating to your `project` in the top left corner, selecting `Settings` in the side nav, and selecting the `API Access Keys` card

ja-translated ![open project settings](/img/builder/builder_open_access_key_settings.png)

MOCKDATA + ja-translated Within the `settings` of the `API Access Keys` select the `dev` access key, which is typically used for development environments, whereas the `prod` key is used for production environments for applications in front of your users.

ja-translated ![settings api keys](/img/builder/builder_settings_api_keys.png)

### MOCKDATA + ja-translated (Optional for Development) Create a Service Account

ja-translated In order to integrate some services from the Sequence stack, a Service Account has to be created to make updates or read from state - including [Collections API](/guides/metadata-guide) or [Webhooks](/api/indexer/examples/webhook-listener) - a `Secret API Key` is required.

MOCKDATA + ja-translated One can navigate to the API Access Keys section by selecting the project, accessing the sidebar nav page of `Settings`, and selecting the `API Access Keys` card

ja-translated ![open project settings](/img/builder/builder_open_access_key_settings.png)

ja-translated And scroll to the bottom, where you will find the `Secret API Access Key(s)` section

MOCKDATA + ja-translated Where you would select `+ Add Service Account`

ja-translated ![secret API keys](/img/builder/builder_service_accounts_create_secret_api_key.png)

MOCKDATA + ja-translated And either create a `Read` key, or, a `Write` secret API access key.

ja-translated For making updates to the Sequence system, select `Write` and `+ Add Service Account` to include in your integration.

MOCKDATA + ja-translated ![create a write key](/img/builder/builder_service_accounts_create_write_key.png)

ja-translated Copy the JWT token and save it somewhere safe, or, in your project

<Callout type="warning">
  MOCKDATA + ja-translated Warning: You won't have access to the generated `Secret API Access Key` in the future from
  the Builder interface, make sure to save it somewhere safe.
</Callout>

MOCKDATA + ja-translated ::::
