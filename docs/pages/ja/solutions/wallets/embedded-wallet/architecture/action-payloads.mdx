# ja-translated Action payloads
MOCKDATA + ja-translated Each write operation in Sequence Embedded Wallet uses a payload. Payloads are JSON objects. They hold all data about the intent of the action and the user's signature.

ja-translated The Embedded Wallet SDK generates Payloads. To complete the action, send them to the Embedded Wallet API.

MOCKDATA + ja-translated The Embedded Wallet API key authenticates all calls to the Embedded Wallet API. This includes calls to the `/sendPayload` endpoint. The Embedded Wallet API key is a secret. Keep it safe and never share it or store it in a client-side application.
:::

## ja-translated Payload structure
MOCKDATA + ja-translated You don't need to manually change the payload contents to integrate Embedded Wallet. But they have strict definitions. Use them if you need to.

ja-translated The payload structure is as follows:

| ja-translated Name                  | MOCKDATA + ja-translated Type   | ja-translated Description                                                      | MOCKDATA + ja-translated Optional |
| ----------------------------------- | ------------------------------- | ------------------------------------------------------------------------------ | --------------------------------- |
| MOCKDATA + ja-translated version    | MOCKDATA + ja-translated string | ja-translated The version of the SDK that generated the payload                | MOCKDATA + ja-translated No       |
| MOCKDATA + ja-translated \* packet  | MOCKDATA + ja-translated object | ja-translated Contents, defined by each payload type                           | MOCKDATA + ja-translated No       |
| MOCKDATA + ja-translated signatures | ja-translated signature\[]      | MOCKDATA + ja-translated Payload signatures by existing sessions               | ja-translated Yes                 |
| ja-translated packet.code           | MOCKDATA + ja-translated string | MOCKDATA + ja-translated Identifier of the payload type                        | MOCKDATA + ja-translated No       |
| ja-translated packet.issued         | MOCKDATA + ja-translated number | MOCKDATA + ja-translated Timestamp of when the payload was issued, in seconds  | MOCKDATA + ja-translated No       |
| ja-translated packet.expires        | MOCKDATA + ja-translated number | MOCKDATA + ja-translated Timestamp of when the payload will expire, in seconds | MOCKDATA + ja-translated No       |

ja-translated \* packet must be organized such that its fields (in JSON structure) appear alphabetically by name.

> MOCKDATA + ja-translated The wallet is optional when openning a new session

#### ja-translated signature type

| ja-translated Name                    | MOCKDATA + ja-translated Type   | ja-translated Description                                    | MOCKDATA + ja-translated Optional |
| ------------------------------------- | ------------------------------- | ------------------------------------------------------------ | --------------------------------- |
| MOCKDATA + ja-translated session      | MOCKDATA + ja-translated string | ja-translated Identifier of the signing session              | MOCKDATA + ja-translated No       |
| MOCKDATA + ja-translated \* signature | MOCKDATA + ja-translated string | ja-translated Signature of the session for the given payload | MOCKDATA + ja-translated No       |

MOCKDATA + ja-translated \* signature should be an EIP-191 signature. See [https://eips.ethereum.org/EIPS/eip-191](https://eips.ethereum.org/EIPS/eip-191)

### ja-translated Open session payload
MOCKDATA + ja-translated Opening a new client-side session on the Embedded Wallet SDK generates this payload. It holds the "proof" that authenticates the user and the session data to link to it.

ja-translated Either `idToken`, `email` or a valid signature _MUST_ be present. If `email` is given, the Embedded Wallet will email the user on their first transaction with this session. The email asks them to confirm it.

MOCKDATA + ja-translated The `wallet` parameter is optional _ONLY_ for this payload. If given, it ensures that the opened session matches the requested wallet.

| ja-translated Name               | MOCKDATA + ja-translated Type   | ja-translated Description                                                   | MOCKDATA + ja-translated Optional |
| -------------------------------- | ------------------------------- | --------------------------------------------------------------------------- | --------------------------------- |
| ja-translated code               | MOCKDATA + ja-translated string | ja-translated The "code" of the payload (`openSession`).                    | MOCKDATA + ja-translated No       |
| MOCKDATA + ja-translated session | MOCKDATA + ja-translated string | MOCKDATA + ja-translated The unique identifier for the session              | MOCKDATA + ja-translated No       |
| ja-translated proof              | MOCKDATA + ja-translated object | MOCKDATA + ja-translated Contains authentication details like JWT and email | MOCKDATA + ja-translated No       |
| ja-translated proof.idToken      | MOCKDATA + ja-translated string | MOCKDATA + ja-translated OAuth2 id token                                    | ja-translated Yes                 |
| ja-translated proof.email        | MOCKDATA + ja-translated string | MOCKDATA + ja-translated User's email address                               | ja-translated Yes                 |

ja-translated Notice that either `idToken`, `email` or a valid signature _MUST_ be present. If `email` is provided, the Embedded Wallet will send an email on the user's first transaction with this session, asking them to confirm it.
