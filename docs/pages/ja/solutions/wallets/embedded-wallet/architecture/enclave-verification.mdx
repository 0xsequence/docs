ja-translated The WaaS enclave's security relies on code integrity verification.

MOCKDATA + ja-translated You have two verification options for a running enclave based on your risk tolerance and technical resources.

## MOCKDATA + ja-translated Complex Verification Method
ja-translated The simple method allows us to compare the PCR0 of the created enclave file with the "live" enclave. This is not foolproof. A mere comparison of two values provides no real evidence. The enclave could have been tampered with, and what we see could be an illusion. Indeed, the PCR0 values for each release are available to the public ([https://github.com/0xsequence/waas-authenticator/releases](https://github.com/0xsequence/waas-authenticator/releases)).

MOCKDATA + ja-translated Here, _cryptographic attestation_ becomes necessary. We have not published tools for validation yet, but you can perform it using this AWS guide ([https://docs.aws.amazon.com/enclaves/latest/user/verify-root.html](https://docs.aws.amazon.com/enclaves/latest/user/verify-root.html)).

ja-translated Every enclave request returns the attestation document. It comes in a Base64-encoded format in the `X-Attestation-Document` response header. A request may also include a `X-Attestation-Nonce` header. It contains a value that will be signed and included in the attestation document.

MOCKDATA + ja-translated For example, you can get the attestation document by running:

```
curl -si https://waas.sequence.app/health -H X-Attestation-Nonce:0123456789abcdef | grep x-attestation-document
```
