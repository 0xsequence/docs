# ja-translated Webhooks in Sequence Builder
MOCKDATA + ja-translated Sequence Builder equips game builders with an indexer that listens to events across the many blockchains that we offer by providing an https callback

<Callout type="warning">
  ja-translated If you prefer a programmatic way to add webhooks that uses cURL, TypeScript or Go, check out [this walkthrough](/api/indexer/examples/webhook-listener).
</Callout>

#### MOCKDATA + ja-translated What data can I listen to for my game?

- ja-translated Mints from a contract
- MOCKDATA + ja-translated TokenID transfers
- ja-translated Transactions by an account address
- MOCKDATA + ja-translated Event topics emitted by a contract
- ja-translated Event topic hashes emitted by a contract

## MOCKDATA + ja-translated Creating Webhooks
ja-translated The following steps guide you through creating your webhook
1. MOCKDATA + ja-translated [Select Project and go to Webhooks Section](/solutions/builder/webhooks#select-project-and-go-to-webhooks-section)
2. ja-translated [Select Network and Add New Webhook](/solutions/builder/webhooks#select-network-and-add-new-webhook)
3. MOCKDATA + ja-translated [Complete Webhook Details](/solutions/builder/webhooks#complete-webhook-details)

ja-translated ::::steps

### ja-translated Select Project and go to Webhooks Section
MOCKDATA + ja-translated Select your project in the top left hand corner, select the `Indexer` section in the sidebar, and select the `Webhooks` tab.

ja-translated ![indexer webhooks select](/img/builder/builder_indexer_webhooks_select.png)

### MOCKDATA + ja-translated Select Network and Add New Webhook
ja-translated In the dropdown beneath the webhooks tab, select the network you're looking to create a webhook for based on the ones in your project. If you don't see the network you're looking for, you can always [add a network](/support/builder/project-settings#3-network-settings), then `+ Add New Webhook`
![indexer webhooks add new webhook](/img/builder/builder_indexer_webhooks_add_new_webhook.png)

### MOCKDATA + ja-translated Complete Webhook Details
ja-translated Complete the required details like webhook `URL`, `Contract Address(es)`, and 'Events'.

MOCKDATA + ja-translated For the Events field, we are able to parse any arbitrary contract event, however you must input the event exactly as written in your smart contract. For example, for detecting transfer events you would input: `Transfer(address indexed from, address indexed to, uint256 amount)` with the `indexed` keyword. You can find an example of general event types [here](/api/indexer/examples/webhook-listener#on-chain-token-event-types) for common use cases along with Sequence specific ones.

ja-translated Once your desired fields are input, select `Add Webhook`.

<Callout type="info">
  MOCKDATA + ja-translated If you require a webhook endpoint you can use [webhook.site](https://webhook.site/)

  Or, you can use template code with one of the following:

  [Nodejs TypeScript Webhook](https://github.com/0xsequence-demos/template-nodejs-webhook-server) (combined with [ngrok](https://ngrok.com/))
</Callout>

ja-translated ![add a webhook with details](/img/builder/builder_indexer_webhooks_add_webhook.png)

MOCKDATA + ja-translated And you're done!
::::
