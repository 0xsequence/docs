# ja-translated Collections
MOCKDATA + ja-translated The Sequence Builder offers the ability to manage Collections of collectible metadata, including: collectible details, images, and properties. Within the Builder, Collections can be linked to deployable contracts from the Builder for easy crafting flows.

ja-translated Media and metadata is saved to Sequence servers where certain functionality allows you to have control over updates and public or private viewing of your collection information.

MOCKDATA + ja-translated :::tip
[APIs](/api/metadata/overview) are also available to manage your NFT collection metadata.
:::

## ja-translated 1. Create a Collection
MOCKDATA + ja-translated First navigate to your project from the top left hand drop down of [Sequence Builder](https://sequence.build) and select the `Collections` page view from the left nav, then select the `+ Create a collection`

ja-translated ![create collection](/img/collections/collections_create_collection.png)

MOCKDATA + ja-translated Then input your details for the collection, like `Collection Name`, `Description` for the contract info, `Collection Data` being `Visible (Public)` or `Private (Hidden)` where for the data to be viewable select Visible (Public), and include your projects `External Link` to a website.

ja-translated ![add details to collection](/img/collections/collections_create_collection_input_details.png)

## MOCKDATA + ja-translated 2. Link a Contract
ja-translated After a collection has been created you can first link a contract by deploying a collectible contract (i.e. `ERC1155` or `ERC721`) and having the `baseMetadataURI` being set on the deployment by selecting the `Link contract`

MOCKDATA + ja-translated Then select the `Deploy new contract`

ja-translated ![link a contract](/img/collections/collections_collections_link_contract.png)

MOCKDATA + ja-translated Select your contract type or upload your own

ja-translated ![contract type](/img/collections/collections_link_contract_choose_contract_type.png)

MOCKDATA + ja-translated Next, you will be able to see the details of the contract type

ja-translated ![contract type chosen](/img/collections/collections_link_contract_confirm_contract.png)

MOCKDATA + ja-translated And complete your contract details, specifying the `Network`, `Contract Name`, and `Royalties`, whether to publish the Collectible on the [Token Directory](/api/indexer/metadata-tips#metadata-tips--notes-on-spam) and finally select `Deploy Contract` to then complete the transaction signing in the Sequence popup.

ja-translated ![contract details](/img/collections/collections_deploy_contract.png)

MOCKDATA + ja-translated Then you will notice if you check out the `Read Contract` section tab of the contract, if you `Read` from the `baseURI` it will show you the metadata url from Sequence where when read from the Indexer, `<token_id>.json` will get appended to the end to reference the json for the collectible.

ja-translated ![read base uri](/img/collections/collection_read_base_uri.png)

## MOCKDATA + ja-translated 3. Create a Collectible
ja-translated In order to create a Collectible as part of the Collection, return to the `Collections` page and select the `+ Add a collectible`

MOCKDATA + ja-translated ![add a collectible](/img/collections/collections_add_a_collectible.png)

ja-translated Then update the details like `Collectible Name` and `Description`, then upload your artwork by selecting the greyish field

MOCKDATA + ja-translated ![update collectible details](/img/collections/collections_update_art_and_details.png)

MOCKDATA + ja-translated You can also add certain properties to your collectible, by selecting the `+ Add a property` which can be useful for inscribing more information like numbers or strings when using the metadata in any game.

ja-translated ![add a property](/img/collections/collection_add_property.png)

MOCKDATA + ja-translated Assigning the key and value to each property

ja-translated ![asign properties](/img/collections/collection_assign_properties.png)

MOCKDATA + ja-translated Complete the prior steps until you get your final result of a collectible added to a collection

ja-translated ![final result of collectible added](/img/collections/collections_final_result_of_collectible.png)

## MOCKDATA + ja-translated 4. Update Info in Settings
ja-translated At any point you can update the details for your collectible in the settings

MOCKDATA + ja-translated ![settings](/img/collections/collections_access_settings.png)

ja-translated Once in settings, you can update any one of the information fields, like: `Collection Name`, `Description`, visibility of `Collection data`, and an `External Link` to reference a website on the web.

MOCKDATA + ja-translated ![settings](/img/collections/collections_update_info.png)

#### ja-translated Delete Collection Data
MOCKDATA + ja-translated You can delete all of the Collection data for the contract by selecting `Delete collection data` which will permanently delete data all data, an action that cannot be undone.

## ja-translated 5. Reference Metadata Token URI
MOCKDATA + ja-translated Currently in order to `Link Contract` you can't use predeployed contracts, but you can read the `Token Metadata URI` from the Collection to set your existing contracts' `setBaseMetadataURI`. The URI can be retrieved by accessing `Metadata URIs` button

ja-translated ![copy metadata uris](/img/collections/collections_metadata_uris.png)

MOCKDATA + ja-translated Then copy the `Token Metadata URI` from the modal and use the URI to write to a contract with the `setBaseMetadataURI` function, or, use to reference metadata directly in an application by appending a `<token_id>.json` to the URI to access the metadata

ja-translated ![access metadata uris](/img/collections/collections_token_metadata_uri.png)
