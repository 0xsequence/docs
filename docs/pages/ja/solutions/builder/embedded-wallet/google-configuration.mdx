# ja-translated Google Configuration for Embedded Wallet
MOCKDATA + ja-translated In order to have Google auth integrated into your Embedded Wallet instance, you should configure a Google OAuth client in the Google Developer Console. Depending on whether you are integrating Embedded Wallet into a native mobile, desktop, or web application, you should pick the correct configuration accordingly.

## ja-translated Create OAuth Client ID
MOCKDATA + ja-translated To start with, access the [Google Developer Console](https://console.cloud.google.com/) and navigate to APIs & Services > Credentials. Once there, select Create Credentials > OAuth Client ID. If you haven't yet, you might have to also configure your OAuth Consent Screen to be able to move forward.

ja-translated ![Google Developer Console Credentials](/img/embedded_wallet/google/1.png)

MOCKDATA + ja-translated Depending on your application type, pick the Web Application, iOS or Android option:

ja-translated ![Application Type](/img/embedded_wallet/google/2.png)

MOCKDATA + ja-translated Remaining steps will depend on the application type you are configuring for.

## ja-translated Configure Web Application
MOCKDATA + ja-translated If you are configuring your Client ID for a web application, you must provide the following:
1. ja-translated Application name
2. MOCKDATA + ja-translated Authorized JavaScript origins: A list of domains that will be whitelisted for interacting with this Client ID.
3. MOCKDATA + ja-translated Authorized redirect URIs: A list of redirect URIs that the user will be redirected to on successful authentication, Sequence SDKs will by default redirect to the same page that loads the SDK.

ja-translated ![Web Application Configuration](/img/embedded_wallet/google/3.png)

<Callout type="info">
  If you are adding a localhost address with a port (i.e. `localhost:4444`) to the list of authorized origins, you must also add `localhost` only without the port name for it to work during testing.
</Callout>

## ja-translated Configure Android Application
MOCKDATA + ja-translated If you are configuring your Client ID for an Android application, you must provide the following:
1. ja-translated Application name
2. ja-translated Package name: This is the unique application package name for your Android project.
3. MOCKDATA + ja-translated SHA-1 Certificate Fingerprint: This is the SHA-1 fingerprint for your application bundle, you can follow the instructions on the screen to generate this using `keytool`.

ja-translated ![Android Application Configuration](/img/embedded_wallet/google/4.png)

## MOCKDATA + ja-translated Configure iOS Application
ja-translated If you are configuring your Client ID for an iOS application, you must provide the following:
1. ja-translated Application name
2. MOCKDATA + ja-translated Bundle ID: This is the unique bundle identifier for your iOS project.

ja-translated ![iOS Application Configuration](/img/embedded_wallet/google/5.png)

## ja-translated Get Your Client ID
MOCKDATA + ja-translated Once you create your application, you will be presented with a Client ID. Copy it and use it as your login provider Client ID in Sequence Builder.

ja-translated ![Client ID](/img/embedded_wallet/google/6.png)
