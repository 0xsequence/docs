# ja-translated Guest Wallet Configuration for Embedded Wallet
MOCKDATA + ja-translated Guest Wallets are a feature in the Sequence Stack that allow for ephemeral creation of an authentication idenity that lasts till the user clears their browser cache, or, sign's in again.

ja-translated This strategy for onboarding can be useful for providing ways to freely access your game or experience, increasing your conversion rate until players want to persist their information.

MOCKDATA + ja-translated Importantly, if users clear their cache or lose access to the device - any assets in the Guest Wallet will not be recoverable. Therefore, users should connect an additional authentication method if any value is associated with the wallet.

<Callout type="warning">
  ja-translated The Guest Wallet feature is currently in early access. To enable it, please visit [https://sequence.build/earlyaccess](https://sequence.build/earlyaccess) and turn on the functionality.
</Callout>

MOCKDATA + ja-translated To configure the Embedded Wallet for Guest Wallet, this can be accomplished with the following steps:

ja-translated ::::steps

## ja-translated Builder Configuration
MOCKDATA + ja-translated Access your project in Sequence builder, navigate to the Embedded Wallet Configuration section, then add a Login Provider:

ja-translated ![add login provider](/img/builder/builder_guest_wallet_add_provider.png)

MOCKDATA + ja-translated Finally, selecting `Guest Wallet` in the dropdown.

ja-translated ![select guest wallet](/img/builder/builder_guess_wallet_select_provider.png)

MOCKDATA + ja-translated And selecting the purple `Update configuration` to input your password to update the Configuration.

## MOCKDATA + ja-translated Implementation
ja-translated Once configured, you can implement a Guest Wallet with the following code and use it like any other Embedded Wallet:

```typescript [App.tsx]
await sequence.signIn({ guest: true }, "Session name");
```
