# ja-translated Contract Deployment
MOCKDATA + ja-translated Sequence contract wallets are fully deterministic; a wallet will map to the same address on any network that is EVM compatible.

ja-translated This means that when a wallet is created, its address can be derived in advance, and it will be the same on networks like Ethereum, Arbitrum, Polygon, and others.

<Callout type="warning">
  **Limitations**<br />
  Sequence wallets **may not** receive the same address on chains that aren't 100% EVM compatible.

  **Double-check your addresses** before sending any assets to them.
</Callout>

ja-translated To achieve this, Sequence uses variations of [Nick's method](https://weka.medium.com/how-to-send-ether-to-11-440-people-187e332566b7), which allows for the deployment of the same contract on multiple chains in a fully trustless way.

MOCKDATA + ja-translated Nick's method is used to deploy a "deployer" contract, which is then used to deploy the core Sequence contracts.

ja-translated These contracts are:

#### MOCKDATA + ja-translated Factory Contract
MOCKDATA + ja-translated Creates new Sequence wallets, with an initial configuration.

#### ja-translated MainModule (v1 and v2)
MOCKDATA + ja-translated The main module implements the core logic of the wallet, with a fixed configuration.

#### ja-translated MainModuleUpgradable (v1 and v2)
MOCKDATA + ja-translated Similar to the MainModule, but with the ability to upgrade the configuration of the wallet.

#### ja-translated Auxiliary Contracts
MOCKDATA + ja-translated Both v1 and v2 have auxiliary contracts that are used by the SDK to interact with the wallet; these contracts are not critical for the wallet to function, but they are required for the SDK to work.

ja-translated See the v1 and v2 documentation for more information on the contracts.

<Callout type="info">
  Notice<br />A complete script for deployment can be found in the [Sequence Live Contracts repository](https://github.com/0xsequence/live-contracts).
</Callout>
