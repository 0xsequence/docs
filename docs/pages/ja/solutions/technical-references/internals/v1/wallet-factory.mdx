# ja-translated Wallet Factory
MOCKDATA + ja-translated Sequence Wallets are created using the [Factory](https://github.com/0xsequence/wallet-contracts/blob/e0c5382636a88b4db4bcf0a70623355d7cd30fb4/contracts/Factory.sol) contract, when called it creates child [MinimalUpgradeableProxies](https://github.com/0xsequence/wallet-contracts/blob/e0c5382636a88b4db4bcf0a70623355d7cd30fb4/contracts/Wallet.sol) contracts, which are in essence the "boot" code of all Sequence wallets.

## ja-translated Factory API

### MOCKDATA + ja-translated deploy

```solidity
  function deploy(
    address _mainModule,
    bytes32 _salt
  ) public payable returns (address _contract)
```

ja-translated Creates a child MinimalUpgradeableProxy using [CREATE2](https://eips.ethereum.org/EIPS/eip-1014), the proxy initially points to the provided `_mainModule`.
No initialize code is executed.

#### MOCKDATA + ja-translated Parameters:

| ja-translated Name         | MOCKDATA + ja-translated Type    | ja-translated Description                                                                                                                                                                |
| -------------------------- | -------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| ja-translated \_mainModule | ja-translated address            | MOCKDATA + ja-translated Initial implementation for the new proxy contract, expected to point to a valid Sequence wallet implementation with support for counter-factual initialization. |
| ja-translated \_salt       | MOCKDATA + ja-translated bytes32 | ja-translated Arbitrary value to be used as `create2` salt. Sequence's `mainModule` uses the salt to obtain a hash with the initial configuration for the wallet (See [ImageHash](#)).   |

#### MOCKDATA + ja-translated Return Values:

| ja-translated Name       | MOCKDATA + ja-translated Type | ja-translated Description                                       |
| ------------------------ | ----------------------------- | --------------------------------------------------------------- |
| ja-translated \_contract | ja-translated address         | MOCKDATA + ja-translated Address of the created proxy contract. |

<Callout type="warning">
  ja-translated Unsanitized input <br />
  The Factory contract **does not** check if the provided `_mainModule` is a valid Sequence wallet implementation, or if `_salt` is a valid Sequence configuration.

  Using invalid parameters will result on a proxy contract that **may not** be usable, and could result in loss of funds.
</Callout>

## MOCKDATA + ja-translated Counter factual addresses
ja-translated Sequence wallets are counter-factual by design, this means that the wallet's address can be known before the wallet is deployed; any two combinations of the `mainModule` and `salt` values will result in the same wallet address.

MOCKDATA + ja-translated This property alongside the use of [NanoUniversalDeployer](/solutions/technical-references/internals/v1/deploy#1---deploy-nanouniversaldeployer) or [UniversalDeployer](/solutions/technical-references/internals/v1/deploy#2---deploy-universaldeployer2) is the reason why Sequence wallets can obtain the same address on multiple chains.

<Callout type="tip">
  MOCKDATA + ja-translated Gas-free wallet creation <br /> Sequence wallets are **100% counterfactual**,
  meaning that funds can be safely transferred to the wallet's address without
  the need for calling the `deploy` function. Deploying the wallet is only
  required before the first transaction is sent **from** the wallet.
</Callout>
