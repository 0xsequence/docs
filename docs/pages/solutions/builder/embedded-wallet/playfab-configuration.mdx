# Playfab Configuration for Embedded Wallet

In order to have PlayFab auth integrated into your Embedded Wallet instance, you have to configure a PlayFab Game in the [Microsoft PlayFab Developer](https://developer.playfab.com/) dashboard, by starting with the following below steps:

::::steps
### Sign Up / Sign In

First access the sign in page [here](https://developer.playfab.com/):

![PlayFab sign in page](/img/guides/playfab/playfab_developer_sign_in.png)

### Create a New Studio 

After you've signed in, go to the [following link to create a studio](https://developer.playfab.com/en-us/r/studio/new) which will generate a `Title ID` for you.

### Retrieve a Title ID

Afer you've created a studio, you can retrieve the `Title ID` to include it in the SDK on the [following page](https://developer.playfab.com/en-us/my-games)

![Retrieve a Title ID](/img/guides/playfab/playfab_developer_retrieve_title_id.png)

Learn more below how to implement a full stack experience with a front-end for your application and to take care of unique identifiers in your experience.
::::

## Create a React Project with Vite with Neccessary Packages

We will start by creating a React project with vite:

```sh
npm create vite
# or
pnpm create vite
# or
yarn create vite
```

## Initialize the Embedded Wallet SDK

Then, start by creating a `config.ts` file within the `/src` directory.

And then, you can obtain a project access key from sequence.build using [this walkthrough](/solutions/builder/getting-started) and a waas config key using [this explainer](https://docs.sequence.xyz/solutions/builder/embedded-wallet):

```ts [config.ts]
import { SequenceWaaS } from '@0xsequence/waas'

const SEQUENCE_PROJECT_ACCESS_KEY = import.meta.env.VITE_SEQUENCE_PROJECT_ACCESS_KEY
const SEQUENCE_WAAS_CONFIG_KEY = import.meta.env.VITE_SEQUENCE_WAAS_CONFIG_KEY

export const sequence = new SequenceWaaS({
    network: 'base-sepolia',
    projectAccessKey: SEQUENCE_PROJECT_ACCESS_KEY,
    waasConfigKey: SEQUENCE_WAAS_CONFIG_KEY
})
```

With the project environment file in the root of the project, called `.env`, instantiated as:

```sh [.env]
VITE_SEQUENCE_PROJECT_ACCESS_KEY=
VITE_SEQUENCE_WAAS_CONFIG_KEY=
VITE_PLAYFAB_TITLE_ID=
```

## Import a PlayFab Client Library
In the html code of the project contained in `index.html`, include the `PlayFabClientApi.js` script file from the PlayFab website:

```html [index.html]
<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <link rel="icon" type="image/svg+xml" href="/vite.svg" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Playfab Sign-in (via Google)</title>
  </head>
  <body>
    <div id="root"></div>
    <!-- add the below line -->
    <script src="https://download.playfab.com/PlayFabClientApi.js"></script>
    <script type="module" src="/src/main.tsx"></script>
  </body>
</html>
```

## Create a Google Handler

For this, you'll need to follow [this configuration guide](/solutions/builder/embedded-wallet/google-configuration/) to obtain a Google Client ID, then add to the following `main.tsx` file:

```ts [main.tsx]
...
import { GoogleOAuthProvider } from '@react-oauth/google'

const googleClientId = '<GOOGLE_CLIENT_ID>'

createRoot(document.getElementById('root')!).render(
  <StrictMode>
    <GoogleOAuthProvider clientId={googleClientId}>
      <App />
    </GoogleOAuthProvider>
  </StrictMode>,
)
```

Next, implement a google function handler that calls the Google Sign In Widget, when on success, then calls the PlayFab Client SDK with the passed in `access_token`, and subsequently passes the `SessionTicket` to the Sequence Embedded Wallet SDK:

```ts [App.tsx]
import { useGoogleLogin } from '@react-oauth/google'

...
const handleGooglePlayfabLogin = useGoogleLogin({
    flow: 'implicit',
    onSuccess: tokenResponse => {
      ;(window as any).PlayFabClientSDK.LoginWithGoogleAccount(
        {
          AccessToken: tokenResponse.access_token, // This access token is generated after a user has signed into Google
          CreateAccount: true,
          TitleId: import.meta.env.VITE_PLAYFAB_TITLE_ID
        },
        async (response?: { data: { SessionTicket: string } }, error?: Error) => {
          if (response) {
            try {
              const seqRes = await sequence.signIn(
                {
                  playFabTitleId: import.meta.env.VITE_PLAYFAB_TITLE_ID,
                  playFabSessionTicket: response.data.SessionTicket
                },
                'playfab session'
              )
              setWalletAddress(seqRes.wallet)
            } catch (e) {
              console.error('Error: ' + JSON.stringify(error))
            }
          } else if (error) {
            console.error('Error: ' + JSON.stringify(error))
          }
        }
      )
    }
  })
```

## Create a Button Handler
Finally, implement within the application, a sign in button to call the handler:

```jsx [App.tsx]
<button onClick={() => handleGooglePlayfabLogin()}>sign in</button>
```

## Error Codes Causes

### 1. `Invalid Verifier`

The Playfab title ID in the builder UI does not match the title ID you have set up in the application.

## Conclusion

By following the Sequence PlayFab configuration and quickstart, you would have created a Game Title in the Microsoft PlayFab dashboard, generated the necessary configuration keys from the Sequence Builder, and created a React application that will accept these environment variables to create a Sequence Embedded Wallet that can perform many different functions with EVM based blockchains.
