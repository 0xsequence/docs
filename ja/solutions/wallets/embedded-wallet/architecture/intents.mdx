「インテント」は、アカウント、セッション、ウォレットとの対話に使用できるアクションです。このドキュメントでは、それらが内部でどのように機能するかを説明します。複雑さを心配する必要はありません - SDKがすべてを処理します。

インテントはJSONオブジェクトと考えてください。実行したいアクションを表し、セッションキーで署名されています。例えば、メッセージに署名したい場合、WaaS APIにインテントを送信します。

## インテントとは何か？

高レベルでは、インテントは単にユーザーが実行しようとするアクションを指定し、セッションキーを使用して署名されたJSONオブジェクトです。例えば、メッセージに署名するために、クライアントは以下のインテントをWaaS APIに送信します：

```
{
    "version": "0.0.0",
    "issuedAt": 1712775116,
    "expiresAt": 1712775416,
    "name": "signMessage",
    "data": {
      "wallet": "0x7B7D7BA79542584f9AeF539F6696c070a4e1Ced6",
      "network": "1",
      "message": "0x48656c6c6f2c20686176652061206e69636520646179"
    },
    "signatures": [
      {
        "sessionId": "0x01047de46debeeb83e1270b80ae65996ea64ad2c033e330cfffe9b725bf078423140124dce0cfe0cab9a61b385ca74fe77cd5e8a7147bfd87052f334cfbadc45c79a",
        "signature": "0x28ec6ddcdbc0987a99dde19792ff0c9be69ad2be873fb9351353171b74d2df3380f0ffcae2b8cb37a0659e374859420500b4399a2aab0209ee7919aaea575ac7"
      }
    ]
}
```

インテントの構造を詳しく見てみましょう。

* **署名：**&#x901A;常、単一の署名がセッションを表します。セッションはローカルで作成されたキーペアです。公開キーは`sessionId`の一部であり、秘密キーはデバイスに残ります。ウォレットへのアクセスを提供するセッションキーは、理想的にはセキュアなオンデバイスエンクレーブに、安全に保管するようにしてください。セッションを使用する前に、WaaS APIに登録する必要があります。
* 署名はインテントの正規表現のEIP-191です。これは、アルファベット順に並べられたキーを持つJSONで、空白を含まず、`signatures`フィールドを除外します。
* **名前とデータ：**&#x540D;前とデータは各インテントによって変わります。名前はインテントの種類を識別し、それに関連するデータを示します。
* **タイムスタンプ：**&#x41;PIはunixタイムスタンプをチェックして、インテントの有効期間を判断し、古いインテントが将来再利用されるのを防ぎます。
