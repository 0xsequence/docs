サインアップとプロジェクトの作成は簡単で、数分もかかりません。Builderへのサインアップと最初のプロジェクトの作成をサポートします。

始めましょう！

## Sequence Builderにサインアップする

<Steps>
  <Step title="Go to sequence.build">
    Sequence.Buildに移動し、[Sequence.Build](https://sequence.build/)をクリックして`Start Building`または`Sign in`ボタンをクリックして始めましょう。

    <Frame>
      ![Sequence builder landing page](/images/builder/builder_landing_page_signup_project.png)
    </Frame>
  </Step>

  <Step title="Choose a sign in method">
    Builderに初めてサインインすると、ウォレットが作成されます。

    ただし、すでにSequenceウォレットをお持ちの場合があります。そのウォレットでサインインしている場合は、それを使用するオプションが表示されます。

    ウォレットがない場合は、いくつかのサインインオプションが提供されます。ソーシャルサインインを使用するか、メールアドレスを使用して始めることができます。

    <Frame>
      ![Sequence builder choose signin](/images/builder/builder_choose_signin_signup_project.png)
    </Frame>
  </Step>

  <Step title="Sign in to sequence.build">
    サインイン方法を選択した後、利用規約が表示されます。`Terms of Use`、`Privacy Policy`、および`Master Services Agreement`を読むには、それぞれのリンクをクリックしてください。

    <Frame>
      ![Sequence builder accept terms](/images/builder/builder_accept_terms_signup_project.png)
    </Frame>

    各項目を読んだ後、続行するためにクリックしてください。

    これでBuilderにサインインできました！
  </Step>

  <Step title="Create a Project">
    Builderでの作業はすべてプロジェクトを中心に行われます - では、プロジェクトを始めましょう。

    プロジェクトを開始する最も簡単な方法は、単に`+ Create Project`をクリックすることです。

    <Frame>
      ![Sequence builder dashboard](/images/builder/builder_dashboard_signup_project.png)
    </Frame>
  </Step>

  <Step title="Enter the project details">
    モーダルが表示され、プロジェクトの詳細を入力できます。プロジェクト名を入力し、必要に応じてプロジェクトのアバターを追加してください。

    次に、デプロイを開始したいネットワークを選択します。これは後で変更することができます。

    <Frame>
      ![Sequence builder enter details](/images/builder/builder_enter_details_signup_project.png)
    </Frame>
  </Step>

  <Step title="Create the project!">
    次に`Create new Project`ボタンをクリックしてください！

    <Frame>
      ![Sequence builder project created](/images/builder/builder_project_created_signup_project.png)
    </Frame>

    完了すると、プロジェクトダッシュボードが表示されます。
  </Step>

  <Step title="Claim an API Access Key">
    最後に、開発を始めるにはBuilderからアクセスキーを取得する必要があります。これは左上の`project`に移動し、サイドナビゲーションで`Settings`を選択し、`API Access Keys`カードを選択することで実行できます

    <Frame>
      ![open project settings](/images/builder/builder_open_access_key_settings.png)
    </Frame>

    の`settings`内で`API Access Keys`から`dev`アクセスキーを選択してください。これは通常、開発環境で使用され、`prod`キーはユーザーの前にあるアプリケーションの本番環境で使用されます。

    <Frame>
      ![settings api keys](/images/builder/builder_settings_api_keys.png)
    </Frame>
  </Step>

  <Step title="(Optional for Development) Create a Service Account">
    Sequence スタックからいくつかのサービスを統合するためには、状態の更新や読み取りを行うためのサービスアカウントを作成する必要があります - これには[Collections API](/guides/metadata-guide)または[Webhooks](/api-references/indexer/examples/webhook-listener)が`Secret API Key`必要です。

    プロジェクトを選択し、サイドバーナビゲーションページの`Settings`にアクセスし、`API Access Keys`カードを選択することで、API アクセスキーのセクションに移動できます

    <Frame>
      ![open project settings](/images/builder/builder_open_access_key_settings.png)
    </Frame>

    下にスクロールすると、`Secret API Access Key(s)`セクションがあります

    ここで選択します`+ Add Service Account`

    <Frame>
      ![secret API keys](/images/builder/builder_service_accounts_create_secret_api_key.png)
    </Frame>

    そして`Read`キー、または`Write`シークレットAPIアクセスキーを作成します。

    Sequenceシステムを更新するには、`Write`を選択し、`+ Add Service Account`を統合に含めます。

    <Frame>
      ![create a write key](/images/builder/builder_service_accounts_create_write_key.png)
    </Frame>

    JWTトークンをコピーして、安全な場所またはプロジェクトに保存してください

    <Warning>
      警告：生成された`Secret API Access Key`には将来Builderインターフェースからアクセスできなくなるため、必ず安全な場所に保存してください。
    </Warning>
  </Step>
</Steps>
