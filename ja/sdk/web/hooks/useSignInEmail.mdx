## インポート

```tsx
import { useSignInEmail } from '@0xsequence/connect'
```

## 使用方法

```tsx
import { useSignInEmail } from '@0xsequence/connect'

function App() {
  const email = useSignInEmail()
  
  return (
    <div>
      <h2>User Information</h2>
      {email ? (
        <div>
          <p>Connected with email: {email}</p>
        </div>
      ) : (
        <p>No email associated with connected wallet</p>
      )}
    </div>
  )
}
```

## 戻り値の型

```tsx
string | null
```

フックは以下を返します：

* ウォレットが接続され、関連付けられたメールアドレスがある場合は`string`メールアドレスを含むオブジェクト
* `null`以下の場合：
  * ウォレットが接続されていない
  * 接続されたウォレットに関連付けられたメールアドレスがない
  * ウォレットが切断された

## 状態管理

フックは以下の方法でメールの状態を管理します：

1. ウォレット接続状態の監視
2. ウォレット接続時に保存されたメールアドレスの取得
3. ウォレット切断時のメールアドレスのクリア
4. ローカルストレージに`WaasSignInEmail`キーを使用してメールアドレスを永続化

## 注意事項

このフックは以下の両方で動作するように設計されています：

* Sequence WaaS (Wallet-as-a-Service) ウォレット
* ユニバーサルウォレットタイプ

このフックはSequenceのストレージシステムと統合され、ウォレットが切断された時に適切なクリーンアップを行いながら、セッション間でのメール永続性を維持します。
