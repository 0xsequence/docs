プライマリーセールスでは、ユーザーへのERC721/1155のミントを容易にするスマートコントラクトを使用します。

一般的に、[Sequence Builderから簡単にデプロイできる](/solutions/collectibles/contracts/deploy-primary-sales-contract)監査済みのプライマリーセールスコントラクトを使用することをお勧めします。

## Unityでのプライマリーセールス

UnityでSequenceセールスコントラクトを使用する場合、`ERC721Sale`と`ERC1155Sale`クラスを使用することをお勧めします。それぞれERC721とERC1155のセールスに対応しています。独自のABIを持つプライマリーセールスコントラクトを使用する場合は、`ERC721Sale`と`ERC1155Sale`のように）C#コントラクトラッパーを構築することをお勧めします。これにより、より簡単でエラーの少ないワークフローが実現できます。このドキュメントでは、プライマリーセールスコントラクトの使用に焦点を当てます。

両クラスは簡単に構築できます：

```
ERC1155Sale sale1155 = new ERC1155Sale(contractAddress);
ERC721Sale sale721 = new ERC721Sale(otherContractAddress);
```

一度[Sequence Builderでプライマリーセールコントラクトを設定](/solutions/collectibles/contracts/deploy-primary-sales-contract)したら、Unityでそれらの詳細を取得したい場合があります。

```
IEthClient client = new SequenceEthClient(chain);
SaleDetails globalSaleDetails = await sale1155.GetGlobalSaleDetailsAsync(client);
SaleDetails tokenSaleDetails = await sale1155.TokenSaleDetailsAsync(client);
SaleDetails nftSaleDetails = await sale721.GetSaleDetailsAsync(client);
```

最後に、チェックアウトするためには、セールコントラクトに価格通貨に対する適切な支出権限を付与し、セールコントラクトのミント関数を呼び出す必要があります（またはネイティブトークンで価格設定している場合は、トランザクションの'value'として支払いを提供します）。

```
List<Transaction> transactions = new List<Transaction>();
ransactions.Add(new RawTransaction(paymentToken.Approve(sale1155.Contract.GetAddress(), tokenSaleDetails.Cost * _amount)));
transactions.Add(new RawTransaction(sale1155.Mint(_wallet.GetWalletAddress(),
    new BigInteger[] { BigInteger.Parse(_tokenId) },
    new BigInteger[] { BigInteger.Parse(_amount.ToString()) }, null, paymentToken,
    tokenSaleDetails.Cost * _amount)));

_wallet.SendTransaction(chain, tranactions.ToArray());
```

<Tip>プライマリーセールスのコントラクトと価格通貨のコントラクトアドレスを[Gas Tank](/solutions/builder/gas-tank)に追加することを忘れないでください。これらのトランザクションをスポンサー化したい場合は！</Tip>
