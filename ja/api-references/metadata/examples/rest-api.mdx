Sequence メタデータ API サービスは、Ethereum互換チェーンのトークンおよびNFTメタデータを照会するためのシンプルで高速なAPIを提供します。

簡単に言えば、メタデータ APIを使用すると、[サポートされている複数のEthereumチェーン](https://status.sequence.info)上のERC20、ERC721、またはERC1155コントラクトのトークンメタデータを照会できます。

また、Sequence メタデータサービスは[Sequence Indexer](/api-references/indexer/overview)に自動的に統合されています。
しかし、以下で示すように、トークン/コントラクトのメタデータを直接照会できることも便利です :)

<Note>
  \[プロのヒント: RPC vs RESTエンドポイント]

  Sequence メタデータサービスは、RPCとRESTの両方のエンドポイントで利用可能です。アプリケーションにはRPCエンドポイントを推奨していますが、利便性のためRESTエンドポイントも利用可能なオプションとして提供しています。

  RPCエンドポイントについては[トークンメタデータ RPC](/api-references/metadata/examples/token-metadata)と[コントラクトメタデータ RPC](/api-references/metadata/examples/contract-metadata)をご覧ください。
</Note>

## 例

Polygonネットワーク上のSkyweaver 1155カードのメタデータを照会したいとしましょう。もちろん、以下の例は任意のネットワーク/コントラクト/トークンの組み合わせでも機能します。

まず、Skyweaver 1155アセットコントラクトはアドレス`0x631998e91476DA5B870D741192fc5Cbc55F5a52E`
にあります。次に、SkyweaverのコントラクトのトークンID`20`を照会するメタデータルックアップエンドポイントを構築しましょう：

[リクエスト](https://metadata.sequence.app/tokens/polygon/0x631998e91476DA5B870D741192fc5Cbc55F5a52E/20)

**レスポンス：**

```json
    "image": "https://assets.skyweaver.net/TNqWLuJZ/webapp/cards/full-cards/6x/20-silver.png",
    "decimals": 2,

      "baseCardId": 20,

      "id": 20,
      "silverCardId": 65556
    },

```

複数のトークンを同時に照会したい場合は、トークンIDをカンマ区切りで含めることができます[https://metadata.sequence.app/tokens/polygon/0x.../20,21,22](https://metadata.sequence.app/tokens/polygon/0x631998e91476DA5B870D741192fc5Cbc55F5a52E/20,21,22)-- クリックしてトークンID`20`、`21`、および`22`の単一のバッチリクエストのJSONレスポンスを確認してください。

上記のmetadata.sequence.app URLを自分のコントラクトや他の人気プロジェクトに変更して、レスポンスがどのように返ってくるか試してみてください。ネットワークは[サポートされているネットワークのいずれか](/)
に変更し、任意のコントラクトやトークンIDを指定できます。

上記の例のような*トークンレベルのメタデータ*を簡単に照会できるだけでなく、*コントラクトレベルのメタデータ*も照会できます。

## 使用方法

メタデータ APIサービスは、シンプルな[RESTインターフェース](#rest-endpoints)、または[RPCクライアント](#rpc-client)でアクセスできます。
RPCクライアントでは、Webブラウザ、Node、Goのクライアントを提供しています。ほとんどの統合にはRPCクライアントを推奨していますが、
利便性のためにRESTインターフェースも提供しています。

## RESTエンドポイント

上記の[例](#example)で述べたように、RESTエンドポイントの一般的な形式は：

```
GET https://metadata.sequence.app/tokens/<network>/<contractAddress>[/<tokenID>]
```

ここで`<network>`は以下のいずれかである必要があります`Chain ID`または`Chain Handle`の[サポートされているネットワーク](/)。

**コントラクトレベルのメタデータを取得：**

```
GET https://metadata.sequence.app/tokens/<network>/<contractAddress>
```

[例](https://metadata.sequence.app/tokens/polygon/0x631998e91476DA5B870D741192fc5Cbc55F5a52E)

**トークンレベルのメタデータを取得：**

```
GET https://metadata.sequence.app/tokens/<network>/<contractAddress>/<tokenID>[,<tokenID>,...]
```

例：

* [トークンID 20](https://metadata.sequence.app/tokens/polygon/0x631998e91476DA5B870D741192fc5Cbc55F5a52E/20)
* [トークンID 20 & 21](https://metadata.sequence.app/tokens/polygon/0x631998e91476DA5B870D741192fc5Cbc55F5a52E/20,21)

## RPCクライアント

メタデータRPCインターフェースは、メタデータサービスの全機能を提供します。

以下のSDKを提供しています：[Web / node.js](https://github.com/0xsequence/sequence.js)および[Go](https://github.com/0xsequence/go-sequence)。
また、別の言語ターゲットでMetadataサービスを統合したい場合は、以下のAPIリファレンスに従って
HTTPリクエストを実装してください。さらに、TypescriptクライアントのソースコードをMetadata RPCクライアントの[参考
実装として読んでください](https://github.com/0xsequence/sequence.js/blob/master/packages/metadata/src/metadata.gen.ts)。

## Metadata RPCメソッド

**Metadata APIエンドポイント：** [https://metadata.sequence.app](https://metadata.sequence.app)

**Metadata RPCメソッド：**

* `GetTokenMetadata` - 単一のコントラクトからERC721またはERC1155トークンのメタデータを取得
* `GetTokenMetadataBatch` - 複数のコントラクトからERC721またはERC1155トークンのメタデータを一括取得
* `GetContractInfo` - ERC20、ERC721、またはERC1155コントラクトアドレスのコントラクトメタデータを取得
* `GetContractInfoBatch` - ERC20、ERC721、またはERC1155コントラクトアドレスのコントラクトメタデータを一括取得
* `SearchContracts` - ERC20、ERC721、およびERC1155トークンの検索条件に基づいてコントラクトメタデータ情報を取得
