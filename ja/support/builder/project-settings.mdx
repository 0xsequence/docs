## 設定オプション
Sequence Builder の設定では、プロジェクトに対してさまざまな操作が行えます。

Sequence Builder の設定セクションは、以下の6つに分類されています。
1. **一般**：[プロジェクト名](/support/builder/project-settings#project-name)や[アバター](/support/builder/project-settings#avatar)の更新
2. **詳細**：[テーマ](/support/builder/project-settings#2-advanced-settings)の変更
3. **コラボレーター**：[コラボレーターの追加](/support/builder/project-settings#add-a-collaborator)や[招待リンクの作成](/support/builder/project-settings#create-an-invite-link)
4. **ネットワーク**：プロジェクトで利用する[ネットワーク](/support/builder/project-settings#4-network-settings)の更新
5. **API アクセスキー**：[キーのコピー](/support/builder/project-settings#copy-your-access-key)や[既存キーの設定変更](/support/builder/project-settings#update-api-access-settings-of-a-key)
6. **請求**：[プロジェクトプランの更新](/support/builder/project-settings#update-your-project-subscription-plan)や[超過利用制限の設定](/support/builder/project-settings#set-your-overage-limit)

設定で最後にできる操作は、[プロジェクトの削除](/support/builder/project-settings#delete-a-project)です。

設定はプロジェクトごとに行うため、まずは特定のプロジェクトの設定画面に移動しましょう。

## Builder で設定画面に移動する方法

#### プロジェクトを選択
Sequence Builder にログインすると、プロジェクトの一覧が表示されます。まだプロジェクトを作成していない場合は、ここで作成してください。

まずはプロジェクトを選択しましょう。

<Frame>
  ![Builder select Project](/images/builder/select_project_from_homepage.png)
</Frame>

#### 設定画面へ移動
プロジェクトのダッシュボードに入ったら、画面左側の `Settings` をクリックします。

<Frame>
  ![Builder select Settings](/images/builder/select_settings_from_nav.png)
</Frame>

設定画面には、このガイドで紹介する6つのセクションがあります。

<Frame>
  ![builder settings](/images/builder/builder_settings.png)
</Frame>

## 1. 一般設定
一般設定を開きます。

<Frame>
  ![General settings](/images/builder/builder_settings_select_general.png)
</Frame>

一般設定では、いくつかのオプションが用意されています。

<Frame>
  ![Builder settings select General](/images/builder/builder_settings_general.png)
</Frame>

#### プロジェクト名
最初のオプションはプロジェクト名の変更です。新しいプロジェクト名を入力し、`Update` ボタンを押して保存してください。

#### アバター
２つ目のオプションはプロジェクトアバターの変更です。ダッシュボード左上に表示されるアバターを変更するには、アバター欄をクリックして画像を選択し、`Update` ボタンを押して保存してください。

## 2. 詳細設定
現在 `詳細設定` でできることは多くありませんが、最も重要なのは `Theme` を `☼`（ライト）と `☾`（ダーク）で切り替えられる点です。

<Frame>
  ![advanced settings](/images/builder/settings_advanced_themes.png)
</Frame>

## 3. コラボレーター設定
どのプロジェクトでもコラボレーターの追加・削除が可能です。

以下の手順に従うか、[コラボレーター招待の動画](/support/builder/project-settings#watch-how-to-invite-collaborators)を先にご覧いただくこともできます。

まずはコラボレーター設定を開きます。

<Frame>
  ![Builder settings select Collaborators](/images/builder/builder_settings_collaborators.png)
</Frame>

コラボレーター設定では、2つの方法でコラボレーターを追加できます。

1つ目は、コラボレーターの情報を直接入力し、アクセスレベルを選択して追加する方法です。

2つ目は、招待リンクを作成する方法です。この場合、アクセスレベル、リンクの有効期間、利用回数の上限（任意）を設定できます。

それぞれの方法を順にご紹介します。

### コラボレーターの追加
コラボレーターを追加するには、[正しいプロジェクト](/support/builder/project-settings#select-a-project)を選択し、[設定](/support/builder/project-settings#go-to-settings)にアクセスし、[コラボレーター](/support/builder/project-settings#2-collaborator-settings)セクションにいることを確認してください。

`Add Collaborator` ボタンをクリックします。

ユーザーアドレス欄にウォレットアドレスを入力し、ロールのドロップダウンから希望するアクセスレベルを選択します。

最後に `Add new Collaborator` ボタンをクリックして完了です。

<Frame>
  ![Builder settings Collaborators Add Collaborator modal](/images/builder/builder_settings_add_collaborator_modal.png)
</Frame>

追加が完了すると、ダッシュボードに新しいコラボレーターが表示され、ウォレットアドレスやアクセスレベルも確認できます。

<Frame>
  ![Builder settings Collaborators confirm collaborator](/images/builder/builder_post_add_collaboration.png)
</Frame>

### 招待リンクの作成
`Create Invite Link` ボタンをクリックすると、次のモーダルが表示されます。

<Frame>
  ![create invite link](/images/builder/builder_settings_create_invite_link.png)
</Frame>

次に、招待リンクで付与したいアクセスレベルを選択します：`Read`、`Write`、または `Admin`。

このリンクで招待できるコラボレーター数を入力します（上限設定は任意です）。

リンクの有効期間を選択します：1日、3日、または1週間。

`Create Link` ボタンをクリックして完了です。

作成後、ダッシュボードに招待リンクが表示され、アクセスレベル、上限、作成日、有効期限などが確認できます。

<Frame>
  ![Builder settings Collaborators confirm invite link](/images/builder/builder_invite_link_confirmation.png)
</Frame>

#### 招待リンクの共有
招待リンクを共有するには、`Copy Link` ボタンをクリックします。

共有したい場所でリンクを貼り付け、コラボレーターに送信してください。

#### 招待リンクの削除
招待リンクを削除するには、`Delete Link` ボタンをクリックします。

削除の確認が表示されるので、`Yes` をクリックしてください。

リンクを削除すると、ダッシュボード右下に成功通知が表示され、リンクは一覧から消えます。

## 4. ネットワーク設定
ネットワーク設定を開きます。

<Frame>
  ![Builder settings select Networks](/images/builder/settings_builder_networks.png)
</Frame>

ネットワーク設定では、Sequenceでサポートされているネットワークの一覧が表示されます。ここで、Node Gatewayを通じてアクセスしたいネットワークや、スマートコントラクトをデプロイしたいネットワークを選択できます。

利用したいネットワークを切り替えたら、`Update`ボタンをクリックしてください。

## 5. APIアクセスキーの設定
APIアクセスキーの設定を開きます。

<Frame>
  ![Builder settings select API Access Keys](/images/builder/builder_settings_select_access_keys.png)
</Frame>

APIアクセスキーのセクションでは、各キーの利用状況の確認、デフォルトキーの設定、特定キーの詳細表示など、さまざまなオプションがダッシュボードで利用できます。

新しいキーの追加や、既存キーの詳細編集も可能です。

各キーごとに、以下のような操作が行えます。
- キーの[名前変更](/support/builder/project-settings#renaming-your-key)や[削除](/support/builder/project-settings#deleting-your-key)
- キーの[コピー](/support/builder/project-settings#copy-your-key)や[コード変更](/support/builder/project-settings#rotate-your-key)
- キーの利用[方法](/support/builder/project-settings#set-service-access)や[場所](/support/builder/project-settings#set-domain-access)の設定

### APIアクセスキーの名前変更・削除
キーの名前変更や削除を行うには、[正しいプロジェクト](/support/builder/project-settings#select-a-project)を選択し、[設定](/support/builder/project-settings#go-to-settings)から[APIアクセスキー](/support/builder/project-settings#4-api-access-keys-settings)セクションに進んでください。

編集したいキーを選択します。

<Frame>
  ![Builder settings API Access Keys select API key for rename](/images/builder/builder_settings_set_dev_key.png)
</Frame>

#### キーの名前を変更する
名前欄に新しい名前を入力してください。

その後、`Update`ボタンをクリックします。

<Frame>
  ![Builder settings API Access Keys rename key](/images/builder/builder_rename_key.png)
</Frame>

#### キーを削除する
左下にある`Delete`ボタンをクリックします。

<Frame>
  ![Builder settings API Access Keys delete key](/images/builder/builder_settings_delete_a_key.png)
</Frame>

削除の確認が表示されるので、`Yes` をクリックしてください。

#### アクセスキーをコピーする
APIキーは他の場所やアプリケーションで利用できるよう設計されています。利用するには、コピーしてご自身のアプリやコードに貼り付けてください。

<Frame>
  ![copy Builder settings API Access Key](/images/builder/builder_settings_copy_key.png)
</Frame>

アクセスキーをコピーするには、Access Key欄の`Copy`をクリックします。

クリック後、ボタンが一時的に`✓ Copied`に変わり、コピー完了の確認が表示されます。

これでプロジェクトのアクセスキーがクリップボードにコピーされました。

#### アクセスキーをローテーションする
万が一キーが漏洩した場合や、プロジェクトのコンテキストに合わせてキーを変更したい場合は、アクセスキーのローテーションを行うことができます。

キーをローテーションしても、プロジェクトの利用履歴は失われません。

これを行うには、Access Key欄の`Rotate`ボタンをクリックしてください。

クリックすると、キーが新しくなり、画面右下に成功通知が表示されます。

<Frame>
  ![Builder settings API Access Keys rotate key](/images/builder/builder_settings_rotated_confirmation.png)
</Frame>

### キーのAPIアクセス設定を更新する
キーのAPIアクセス設定を更新するには、[正しいプロジェクト](/support/builder/project-settings#select-a-project)を選択し、[設定](/support/builder/project-settings#go-to-settings)から[APIアクセスキー](/support/builder/project-settings#4-api-access-keys-settings)セクションに進んでください。

アクセス設定を更新したいキーを選択します。

<Frame>
  ![Builder settings API Access Keys select API key for rename](/images/builder/builder_settings_set_dev_key.png)
</Frame>

#### ドメインアクセスの設定
APIキーに紐づくサービスを、ご自身のゲームやアプリに関連するドメインでのみ利用できるように、許可するドメインを指定できます。

これを行うには、アクセスを許可したいドメインをドメイン欄に入力します。

別のドメインも追加したい場合は、`Add Domain`ボタンをクリックして追加してください。これで、キーが利用できるドメインのリストが作成されます。

必要なドメインをすべて追加するまで、このリストに追加を続けてください。

`Update button`をクリックして、設定を保存してください。

<Frame>
  ![Builder settings API Access Keys add domains](/images/builder/builer_settings_set_domain.png)
</Frame>

#### サービスアクセスの設定
APIキーで利用できるサービスを、すべてではなく特定のサービスのみに限定することができます。

画面右下付近に、チェックボックス付きで複数のサービスが表示されています。

キーで利用したいサービスごとにチェックを入れてください。

`Update button`をクリックして、設定を保存してください。

<Frame>
  ![Builder settings API Access Keys check services](/images/builder/builder_settings_update_services.png)
</Frame>

## 6. 請求設定
請求設定を開きます。

<Frame>
  ![Builder settings select Billing](/images/builder/builder_settings_select_billing.png)
</Frame>

ここでは、必要なサブスクリプションプラン情報がすべて確認できます。

[サブスクリプションの更新](/support/builder/project-settings#update-your-project-subscription-plan)や、[超過利用上限の設定](/support/builder/project-settings#set-your-overage-limit)も可能です。

#### 請求情報
請求セクションでは、サブスクリプションプランに関する主な情報をすぐに確認できます。
- 現在のサブスクリプションプラン
- 現在の請求期間の日付
- 月額請求金額
- 次回更新日
- 今月発行されたクレジット数
- 今月利用したクレジット数
- 残りのクレジット数
- 超過利用設定の状況

### プロジェクトのサブスクリプションプランを更新する
プロジェクトのサブスクリプションプランを更新するには、[正しいプロジェクト](/support/builder/project-settings#select-a-project)を選択し、[設定](/support/builder/project-settings#go-to-settings)から[請求](/support/builder/project-settings#5-billing-settings)セクションに進んでください。

`Upgrade Plan`ボタンをクリックします。

<Frame>
  ![Builder settings billing select upgrade plan](/images/builder/builder_settings_upgrade_plan.png)
</Frame>

プランのダッシュボードが表示されるので、各プランを確認し、ご自身に最適なものを選択してください。

希望のプランが決まったら、`Select Plan`ボタンをクリックします。

<Frame>
  ![Builder settings billing upgrade plan dashboard](/images/builder/builder_settings_select_plan.png)
</Frame>

チェックアウト画面に進み、支払い情報を入力します。

<Frame>
  ![Builder settings billing checkout](/images/builder/builder_settings_upgrade_plan_checkout.png)
</Frame>

### 超過利用上限の設定
超過利用上限を設定するには、[正しいプロジェクト](/support/builder/project-settings#select-a-project)を選択し、[設定](/support/builder/project-settings#go-to-settings)から[請求](/support/builder/project-settings#5-billing-settings)セクションに進んでください。また、超過利用上限の設定には有料プランが必要です。

有料プランの場合、まず歯車アイコンのボタンをクリックします。

<Frame>
  ![Builder settings billing set overage](/images/builder/builder_settings_set_overage_limit.png)
</Frame>

`Set overage limit`オプションを切り替えます。

設定したい最大超過金額を入力してください。

`Save changes`ボタンをクリックして完了です。

## プロジェクトの削除

<Warning>
  以下の操作は取り消しできませんので、本当にプロジェクトを削除する場合のみ実施してください。
</Warning>

プロジェクトを削除するには、[正しいプロジェクト](/support/builder/project-settings#select-a-project)を選択し、[設定](/support/builder/project-settings#go-to-settings)セクションに進んでください。また、削除には管理者権限が必要です。

管理者の場合、すべての設定セクションの下部に<span style={{color: "#c2501f"}}>危険ゾーン</span>と表示されたエリアがあります。

このエリアに`Delete this Project`というオプションがあるので、クリックしてください。

<Frame>
  ![Builder delete project](/images/builder/settings_builder_delete_project.png)
</Frame>

この操作を行うと、プロジェクト削除の確認モーダルが表示されます。

<Frame>
  ![Builder プロジェクト削除](/images/builder/builder_settings_input_delete_project_name.png)
</Frame>

プロジェクト名を入力し、`Delete Permanently` ボタンをクリックしてください。
