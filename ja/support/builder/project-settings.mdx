## 設定オプション

Sequence Builderの設定では、プロジェクトに関する様々なアクションが利用可能です。

Sequence Builderの設定セクションは6つのカテゴリーに分かれています：

1. **一般**: [プロジェクト名](/support/builder/project-settings#project-name)または[アバター](/support/builder/project-settings#avatar)
2. **詳細**: [テーマ](/support/builder/project-settings#2-advanced-settings)
3. **共同作業者**: [共同作業者](/support/builder/project-settings#add-a-collaborator)または[招待リンク](/support/builder/project-settings#create-an-invite-link)
4. **ネットワーク**: [ネットワーク](/support/builder/project-settings#4-network-settings)の更新
5. **APIアクセスキー**: [キー](/support/builder/project-settings#copy-your-access-key)と[既存のキーの設定](/support/builder/project-settings#update-api-access-settings-of-a-key)
6. **課金**: [プロジェクトプラン](/support/builder/project-settings#update-your-project-subscription-plan)または[超過制限](/support/builder/project-settings#set-your-overage-limit)

設定で実行できる最後のアクションは[プロジェクトの削除](/support/builder/project-settings#delete-a-project)です。

設定はプロジェクトごとに構成されるため、まずは特定のプロジェクトの設定に移動しましょう。

## Builderでの設定へのアクセス

#### プロジェクトの選択

Sequence Builderにログインすると、プロジェクトのリストが表示されます。まだプロジェクトを作成していない場合は、今作成してください。

プロジェクトを選択することから始めます。

<Frame>
  ![Builder select Project](/images/builder/select_project_from_homepage.png)
</Frame>

#### 設定へ移動

プロジェクトダッシュボードに入ったら、画面左側の`Settings`をクリックします。

<Frame>
  ![Builder select Settings](/images/builder/select_settings_from_nav.png)
</Frame>

設定内には、このガイドで説明する6つのセクションがあります。

<Frame>
  ![builder settings](/images/builder/builder_settings.png)
</Frame>

## 1. 一般設定

一般設定を開きます。

<Frame>
  ![General settings](/images/builder/builder_settings_select_general.png)
</Frame>

一般設定には、いくつかのオプションがあります。

<Frame>
  ![Builder settings select General](/images/builder/builder_settings_general.png)
</Frame>

#### プロジェクト名

最初のオプションはプロジェクト名の変更です。変更するには、フィールドに新しいプロジェクト名を入力し、`Update`ボタンを押して保存します。

#### アバター

2番目のオプションでは、ダッシュボード上のプロジェクトの左上に表示されるプロジェクトアバターを変更できます。アバターフィールドをクリックして画像を選択し、`Update`ボタンを押して変更を保存します。

## 2. 詳細設定

現在、詳細設定では多くのことはできませんが、最も重要なのは`Advanced settings`テーマを`Theme`ライト`☼`ダーク`☾`の間で切り替えられることです

## 3. 共同作業者設定

プロジェクトの共同作業者は追加および削除が可能です。

以下のセクションに従うか、共同作業者を招待する[動画](/support/builder/project-settings#watch-how-to-invite-collaborators)にスキップすることができます。

まず、共同作業者設定を開きます。

<Frame>
  ![Builder settings select Collaborators](/images/builder/builder_settings_collaborators.png)
</Frame>

共同作業者設定では、プロジェクトに共同作業者を追加する2つの方法があります。

1つ目の方法は、情報を入力してアクセスレベルを選択することで、直接共同作業者を追加する方法です。

2つ目の方法は、招待リンクを作成する方法です。このオプションを使用する場合、アクセスレベル、リンクの有効期限、オプションの使用制限を指定できます。

それぞれの方法について見ていきましょう。

### 共同作業者の追加

共同作業者を追加するには、[正しいプロジェクト](/support/builder/project-settings#select-a-project)を選択し、[設定](/support/builder/project-settings#go-to-settings)にアクセスし、[共同作業者](/support/builder/project-settings#2-collaborator-settings)セクションにいることを確認してください。

ボタンをクリックして開始します`Add Collaborator`。

ユーザーアドレスフィールドにウォレットアドレスを入力し、ロールドロップダウンを使用して希望するアクセスレベルを選択します。

最後に`Add new Collaborator`ボタンをクリックします。

<Frame>
  ![Builder settings Collaborators Add Collaborator modal](/images/builder/builder_settings_add_collaborator_modal.png)
</Frame>

確認のため、ダッシュボードに新しい共同作業者のウォレットアドレスとアクセスレベルが表示されるはずです。

<Frame>
  ![Builder settings Collaborators confirm collaborator](/images/builder/builder_post_add_collaboration.png)
</Frame>

### 招待リンクを作成する

まず`Create Invite Link`ボタンをクリックすると、以下のモーダルが表示されます

<Frame>
  ![create invite link](/images/builder/builder_settings_create_invite_link.png)
</Frame>

次に、招待リンクに付与したいアクセスレベルを選択します：`Read`、`Write`、または`Admin`。

このリンクで制限したい共同作業者の数を入力します（制限の設定はオプションです）。

最後に、このリンクの有効期限を選択します：1日、3日、または1週間。

最後に`Create Link`ボタンをクリックします。

確認のため、ダッシュボードに招待リンクが表示され、アクセスレベル、制限、作成日、有効期限までの残り時間が含まれているはずです。

<Frame>
  ![Builder settings Collaborators confirm invite link](/images/builder/builder_invite_link_confirmation.png)
</Frame>

#### 招待リンクを共有する

招待リンクを共有するには、`Copy Link`ボタンをクリックします。

リンクを共有したい場所に移動し、リンクを貼り付けて共同作業者に送信します。

#### 招待リンクを削除する

招待リンクを削除するには、`Delete Link`ボタンをクリックします。

削除の確認を求められます。`Yes`をクリックして削除します。

リンクを削除すると、ダッシュボードの右下に成功通知が表示され、リンクは表示されなくなります。

## 5. APIアクセスキー設定

APIアクセスキー設定を開きます。

<Frame>
  ![Builder settings select API Access Keys](/images/builder/builder_settings_select_access_keys.png)
</Frame>

APIアクセスキーセクションでは、各キーの使用状況の監視、デフォルトキーの設定、特定のキーの詳細表示など、いくつかのオプションを含むダッシュボードが表示されます。

また、新しいキーを追加したり、既存のキーの詳細を編集したりすることもできます。

各キーについて、以下のような特定のアクションを実行できます：

* [名前変更](/support/builder/project-settings#renaming-your-key)または[削除](/support/builder/project-settings#deleting-your-key)する
* [コピー](/support/builder/project-settings#copy-your-key)または[変更](/support/builder/project-settings#rotate-your-key)する
* 設定[方法](/support/builder/project-settings#set-service-access)および[場所](/support/builder/project-settings#set-domain-access)を設定する

### APIアクセスキーの名前変更または削除

キーの名前を変更または削除するには、[正しいプロジェクト](/support/builder/project-settings#select-a-project)を選択し、[設定](/support/builder/project-settings#go-to-settings)にアクセスし、[APIアクセスキー](/support/builder/project-settings#4-api-access-keys-settings)セクションにいることを確認してください。

編集したいキーを選択することから始めます。

<Frame>
  ![Builder settings API Access Keys select API key for rename](/images/builder/builder_settings_set_dev_key.png)
</Frame>

#### キーの名前変更

名前フィールドに新しい名前を入力します。

その後`Update`ボタンをクリックします。

<Frame>
  ![Builder settings API Access Keys rename key](/images/builder/builder_rename_key.png)
</Frame>

#### キーの削除

左下の`Delete`ボタンをクリックします。

<Frame>
  ![Builder settings API Access Keys delete key](/images/builder/builder_settings_delete_a_key.png)
</Frame>

削除の確認を求められます。`Yes`をクリックして削除します。

#### アクセスキーをコピーする

APIキーは別の場所やアプリケーションで使用するように設計されています。使用するには、キーをコピーしてアプリケーションやコードに貼り付ける必要があります。

<Frame>
  ![copy Builder settings API Access Key](/images/builder/builder_settings_copy_key.png)
</Frame>

アクセスキーをコピーするには、`Copy`アクセスキーフィールドに

クリック後、ボタンが一時的に`✓ Copied`に変わり、確認が表示されます。

プロジェクトのアクセスキーがクリップボードにコピーされました。

#### アクセスキーのローテーション

キーが漏洩した場合や、プロジェクトのコンテキストとして何が生成されるかを確認したい場合など、アクセスキーを変更したい場合があります

キーをローテーションしても、プロジェクトの使用履歴は失われません

これを行うには、`Rotate`ボタンをアクセスキーフィールドでクリックします

クリック後、キーが変更され、右下に成功通知が表示されます。

<Frame>
  ![Builder settings API Access Keys rotate key](/images/builder/builder_settings_rotated_confirmation.png)
</Frame>

### キーのAPIアクセス設定の更新

キーのAPIアクセス設定を更新するには、[正しいプロジェクト](/support/builder/project-settings#select-a-project)を選択し、[設定](/support/builder/project-settings#go-to-settings)にアクセスし、[APIアクセスキー](/support/builder/project-settings#4-api-access-keys-settings)セクションにいることを確認してください。

アクセスを更新したいキーを選択することから始めます。

<Frame>
  ![Builder settings API Access Keys select API key for rename](/images/builder/builder_settings_set_dev_key.png)
</Frame>

#### ドメインアクセスの設定

APIキーにリンクされたサービスが、ゲームやアプリに接続されたドメインでのみ利用されるように、キーを使用できる特定のドメインを有効にすることができます。

これを行うには、アクセスを有効にしたいドメインをドメインフィールドに入力します。

別のドメインを追加したい場合は、`Add Domain`ボタンをクリックして追加します。これで、キーを使用できるドメインのリストができました。

必要なすべてのドメインを含めるまで、このリストに追加を続けてください。

更新を完了するには、`Update button`をクリックして保存します。

<Frame>
  ![Builder settings API Access Keys add domains](/images/builder/builer_settings_set_domain.png)
</Frame>

#### サービスアクセスの設定

APIキーを設定して、提供するすべてのサービスではなく、特定のサービスのみを有効にすることができます。

これを行うには、右下付近にチェックボックス付きで表示されている複数のサービスを確認します。

キーにアクセスを許可したい各サービスのボックスにチェックを入れます。

更新を完了するには、`Update button`をクリックして保存します。

<Frame>
  ![Builder settings API Access Keys check services](/images/builder/builder_settings_update_services.png)
</Frame>

## 6. 請求設定

請求設定を開きます。

<Frame>
  ![Builder settings select Billing](/images/builder/builder_settings_select_billing.png)
</Frame>

ここでは、必要なすべてのサブスクリプションプラン情報が提供されます。

また、[サブスクリプションを更新](/support/builder/project-settings#update-your-project-subscription-plan)し、[超過制限を設定](/support/builder/project-settings#set-your-overage-limit)することもできます。

#### 請求情報

請求セクションでは、サブスクリプションプランに関する重要な情報に素早くアクセスできます：

* 現在のサブスクリプションプラン
* 現在の請求期間の日付
* 月額請求金額
* 次回更新日
* 月間発行クレジット
* 現在の期間で使用したクレジット
* 残りのクレジット
* 超過設定のステータス

### プロジェクトのサブスクリプションプランを更新

プロジェクトのサブスクリプションプランを更新するには、[正しいプロジェクト](/support/builder/project-settings#select-a-project)を選択し、[設定](/support/builder/project-settings#go-to-settings)にアクセスし、[請求](/support/builder/project-settings#5-billing-settings)セクションにいることを確認してください。

まず、`Upgrade Plan`ボタンをクリックします。

<Frame>
  ![Builder settings billing select upgrade plan](/images/builder/builder_settings_upgrade_plan.png)
</Frame>

これにより、プラン一覧ダッシュボードに移動し、プランを確認して最適なものを選択できます。

希望のプランが決まったら、`Select Plan`ボタンをクリックします。

<Frame>
  ![Builder settings billing upgrade plan dashboard](/images/builder/builder_settings_select_plan.png)
</Frame>

これにより、支払い情報を入力できるチェックアウト画面に移動します。

<Frame>
  ![Builder settings billing checkout](/images/builder/builder_settings_upgrade_plan_checkout.png)
</Frame>

### 超過制限の設定

超過制限を設定するには、[正しいプロジェクト](/support/builder/project-settings#select-a-project)を選択し、[設定](/support/builder/project-settings#go-to-settings)にアクセスし、[請求](/support/builder/project-settings#5-billing-settings)セクションにいることを確認してください。超過制限を設定するには有料プランが必要です。

有料プランでは、歯車アイコンのボタンをクリックして開始します。

<Frame>
  ![Builder settings billing set overage](/images/builder/builder_settings_set_overage_limit.png)
</Frame>

トグルを切り替えて`Set overage limit`オプションを選択します。

次に、設定したい最大超過額を入力します。

最後に`Save changes`ボタンをクリックします。

## プロジェクトの削除

<Warning>
  以下の手順は破壊的で元に戻すことができないため、プロジェクトを削除する意図がある場合以外は、これらの手順に従わないでください。
</Warning>

プロジェクトを削除するには、[正しいプロジェクト](/support/builder/project-settings#select-a-project)を選択し、[設定](/support/builder/project-settings#go-to-settings)セクションにいることを確認してください。また、プロジェクトを削除するには管理者レベルの権限が必要です。

管理者の場合、すべての設定セクションの下に<span style={{color: "#c2501f"}}>危険ゾーン</span>というセクションが表示されます。

このエリアには、`Delete this Project`というオプションがあります — このオプションをクリックしてください。

<Frame>
  ![Builder delete project](/images/builder/settings_builder_delete_project.png)
</Frame>

この操作により、プロジェクトを削除することを確認するモーダルが表示されます。

<Frame>
  ![Builder delete project](/images/builder/builder_settings_input_delete_project_name.png)
</Frame>

プロジェクト名を入力し、`Delete Permanently`ボタンをクリックします。
